## 11.1 排序算法简介

**排序算法（Sorting Algorithm）** 用于对一组数据按照特定顺序进行排列。排序算法有着广泛的应用，因为有序数据通常能够被更高效地查找、分析和处理。

### 11.1.1 评价维度

| 维度 | 说明 |
|------|------|
| **运行效率** | 时间复杂度尽量低，总体操作数量少（常数项小） |
| **就地性** | 原地排序在原数组上直接操作，无须借助额外辅助数组，节省内存 |
| **稳定性** | 稳定排序后，相等元素的相对顺序不发生改变 |
| **自适应性** | 能利用输入数据已有的顺序信息，最佳时间复杂度优于平均 |
| **基于比较** | 基于比较的排序依赖 `<、=、>` 判断，相对通用；非比较排序可达 O(n) |

### 11.1.2 理想排序算法

运行快、原地、稳定、自适应、通用性好。显然，迄今为止尚未发现兼具以上所有特性的排序算法。因此，在选择排序算法时，需要根据具体的数据特点和问题需求来决定。

## 11.2 选择排序（Selection Sort）

**选择排序**的工作原理非常简单：开启一个循环，每轮从未排序区间选择最小的元素，将其放到已排序区间的末尾。

```python
def selection_sort(nums: list[int]):
    """ 选择排序"""
    n = len(nums)
    # 外循环：未排序区间为 [i, n-1]
    for i in range(n - 1):
        # 内循环：找到未排序区间内的最小元素
        k = i
        for j in range(i + 1, n):
            if nums[j] < nums[k]:
                k = j  # 记录最小元素的索引
        # 将该最小元素与未排序区间的首个元素交换
        nums[i], nums[k] = nums[k], nums[i]
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n²)，非自适应排序 |
| 空间复杂度 | O(1)，原地排序 |
| 稳定性 | ❌ 非稳定排序 |

## 11.3 冒泡排序（Bubble Sort）

**冒泡排序**通过连续地比较与交换相邻元素实现排序。这个过程就像气泡从底部升到顶部一样，因此得名。

### 基本实现

```python
def bubble_sort(nums: list[int]):
    """ 冒泡排序"""
    n = len(nums)
    # 外循环：未排序区间为 [0, i]
    for i in range(n - 1, 0, -1):
        # 内循环：将未排序区间 [0, i] 中的最大元素交换至最右端
        for j in range(i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
```

### 优化版本（提前返回）

```python
def bubble_sort_with_flag(nums: list[int]):
    """ 冒泡排序（标志位优化）"""
    n = len(nums)
    for i in range(n - 1, 0, -1):
        flag = False  # 初始化标志位
        for j in range(i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
                flag = True
        if not flag:
            break  # 此轮未交换任何元素，直接跳出
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n²)，优化后最佳可达 O(n) |
| 空间复杂度 | O(1)，原地排序 |
| 稳定性 | ✅ 稳定排序 |

## 11.4 插入排序（Insertion Sort）

**插入排序**的工作原理与手动整理一副牌的过程非常相似。在未排序区间选择一个基准元素，将其与左侧已排序区间的元素逐一比较大小，并插入到正确位置。

```python
def insertion_sort(nums: list[int]):
    """ 插入排序"""
    # 外循环：已排序区间为 [0, i-1]
    for i in range(1, len(nums)):
        base = nums[i]
        j = i - 1
        # 内循环：将 base 插入到已排序区间 [0, i-1] 中的正确位置
        while j >= 0 and nums[j] > base:
            nums[j + 1] = nums[j]  # 将 nums[j] 向右移动一位
            j -= 1
        nums[j + 1] = base  # 将 base 赋值到正确位置
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n²)，自适应排序，最佳 O(n) |
| 空间复杂度 | O(1)，原地排序 |
| 稳定性 | ✅ 稳定排序 |

### 为什么插入排序更快？

- 冒泡排序基于元素交换，需要借助临时变量（3个单元操作）
- 插入排序基于元素赋值，仅需1个单元操作
- 在数据量较小时，n² 和 nlog n 数值接近，单元操作数量起决定性作用
- Java 内置排序函数采用插入排序：长数组用快速排序，短数组用插入排序

## 11.5 快速排序（Quick Sort）

**快速排序**是一种基于分治策略的排序算法，运行高效，应用广泛。

### 哨兵划分（Partition）

核心操作：选择"基准数"，将所有小于基准数的元素移到左侧，大于的移到右侧。

```python
def partition(nums: list[int], left: int, right: int) -> int:
    """ 哨兵划分"""
    # 以 nums[left] 为基准数
    i, j = left, right
    while i < j:
        while i < j and nums[j] >= nums[left]:
            j -= 1  # 从右向左找首个小于基准数的元素
        while i < j and nums[i] <= nums[left]:
            i += 1  # 从左向右找首个大于基准数的元素
        # 元素交换
        nums[i], nums[j] = nums[j], nums[i]
    # 将基准数交换至两子数组的分界线
    nums[i], nums[left] = nums[left], nums[i]
    return i  # 返回基准数的索引
```

### 完整快速排序

```python
def quick_sort(nums: list[int], left: int, right: int):
    """ 快速排序"""
    # 子数组长度为 1 时终止递归
    if left >= right:
        return
    # 哨兵划分
    pivot = partition(nums, left, right)
    # 递归左子数组、右子数组
    quick_sort(nums, left, pivot - 1)
    quick_sort(nums, pivot + 1, right)
```

### 基准数优化（三数取中）

```python
def median_three(nums: list[int], left: int, mid: int, right: int) -> int:
    """ 选取三个候选元素的中位数"""
    l, m, r = nums[left], nums[mid], nums[right]
    if (l <= m <= r) or (r <= m <= l):
        return mid
    if (m <= l <= r) or (r <= l <= m):
        return left
    return right

def partition_median(nums: list[int], left: int, right: int) -> int:
    """ 哨兵划分（三数取中值）"""
    med = median_three(nums, left, (left + right) // 2, right)
    nums[left], nums[med] = nums[med], nums[left]  # 将中位数交换至最左端
    # 后续与标准 partition 相同...
```

### 递归深度优化

```python
def quick_sort_optimized(nums: list[int], left: int, right: int):
    """ 快速排序（递归深度优化）"""
    while left < right:
        pivot = partition(nums, left, right)
        # 仅对较短的子数组递归
        if pivot - left < right - pivot:
            quick_sort_optimized(nums, left, pivot - 1)
            left = pivot + 1
        else:
            quick_sort_optimized(nums, pivot + 1, right)
            right = pivot - 1
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)，最差 O(n²) |
| 空间复杂度 | O(n)，优化后 O(log n) |
| 稳定性 | ❌ 非稳定排序 |

### 快速排序为什么快？

1. **最差情况概率很低**：绝大多数情况下能在 O(n log n) 下运行
2. **缓存使用效率高**：整个子数组可加载到缓存，访问效率高
3. **常数系数小**：比较、赋值、交换操作总量最少

> ⚠️ **注意**：哨兵划分必须先"从右往左查找"再"从左往右查找"，否则会导致划分失败。

## 11.6 归并排序（Merge Sort）

**归并排序**基于分治策略，包含"划分"和"合并"两个阶段：

1. **划分阶段**：递归地将数组从中点处分开
2. **合并阶段**：从底至顶地将左右两个有序数组合并为一个有序数组

```python
def merge(nums: list[int], left: int, mid: int, right: int):
    """ 合并左子数组和右子数组"""
    tmp = [0] * (right - left + 1)
    i, j, k = left, mid + 1, 0
    # 当左右子数组都还有元素时，进行比较并复制到临时数组
    while i <= mid and j <= right:
        if nums[i] <= nums[j]:
            tmp[k] = nums[i]
            i += 1
        else:
            tmp[k] = nums[j]
            j += 1
        k += 1
    # 复制剩余元素
    while i <= mid:
        tmp[k] = nums[i]
        i += 1
        k += 1
    while j <= right:
        tmp[k] = nums[j]
        j += 1
        k += 1
    # 将临时数组复制回原数组
    for k in range(len(tmp)):
        nums[left + k] = tmp[k]

def merge_sort(nums: list[int], left: int, right: int):
    """ 归并排序"""
    if left >= right:
        return  # 子数组长度为 1 时终止
    mid = (left + right) // 2
    merge_sort(nums, left, mid)
    merge_sort(nums, mid + 1, right)
    merge(nums, left, mid, right)
```

> 归并排序与二叉树后序遍历的递归顺序一致。

### 链表排序优化

对于链表，归并排序可将空间复杂度优化至 O(1)：
- 划分阶段：用迭代替代递归，省去栈帧空间
- 合并阶段：仅改变指针引用，无须创建额外数组

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)，非自适应 |
| 空间复杂度 | O(n)，非原地排序 |
| 稳定性 | ✅ 稳定排序 |

## 11.7 堆排序（Heap Sort）

**堆排序**基于堆数据结构实现。阅读本节前请确保已学完[第8章 堆](./16-堆.md)。

### 算法流程

1. 输入数组并建立大顶堆，最大元素位于堆顶
2. 将堆顶元素与堆底元素交换，堆长度减1
3. 从堆顶开始执行堆化（sift_down）
4. 循环执行第2、3步 n-1 轮

```python
def sift_down(nums: list[int], n: int, i: int):
    """ 堆的长度为 n，从节点 i 开始从顶至底堆化"""
    while True:
        l = 2 * i + 1
        r = 2 * i + 2
        ma = i
        if l < n and nums[l] > nums[ma]:
            ma = l
        if r < n and nums[r] > nums[ma]:
            ma = r
        if ma == i:
            break
        nums[i], nums[ma] = nums[ma], nums[i]
        i = ma

def heap_sort(nums: list[int]):
    """ 堆排序"""
    # 建堆操作：堆化除叶节点以外的所有节点
    for i in range(len(nums) // 2 - 1, -1, -1):
        sift_down(nums, len(nums), i)
    # 从堆中提取最大元素，循环 n-1 轮
    for i in range(len(nums) - 1, 0, -1):
        nums[0], nums[i] = nums[i], nums[0]
        sift_down(nums, i, 0)
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)，非自适应 |
| 空间复杂度 | O(1)，原地排序 |
| 稳定性 | ❌ 非稳定排序 |

## 11.8 桶排序（Bucket Sort）

**桶排序**是分治策略的典型应用。通过设置具有大小顺序的桶，将数据分配到各个桶中，分别排序后合并。

```python
def bucket_sort(nums: list[float]):
    """ 桶排序"""
    k = len(nums) // 2  # 初始化 k 个桶
    buckets = [[] for _ in range(k)]
    
    # 1. 将数组元素分配到各个桶中
    for num in nums:
        i = int(num * k)  # 使用 num * k 映射到索引范围 [0, k-1]
        buckets[i].append(num)
    
    # 2. 对各个桶执行排序
    for bucket in buckets:
        bucket.sort()
    
    # 3. 遍历桶合并结果
    i = 0
    for bucket in buckets:
        for num in bucket:
            nums[i] = num
            i += 1
```

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n + k)，最差 O(n²) |
| 空间复杂度 | O(n + k)，非原地排序 |
| 稳定性 | 取决于桶内排序算法 |

### 如何实现平均分配？

1. **递归划分**：将数据粗略分到3个桶，再将元素较多的桶继续划分
2. **概率分布**：根据数据特点采用概率模型设置分界线

## 11.9 计数排序（Counting Sort）

**计数排序**通过统计元素数量来实现排序，通常应用于整数数组。本质上是桶排序在整型数据下的特例。

### 简单实现

```python
def counting_sort_naive(nums: list[int]):
    """ 计数排序（简单实现）"""
    m = max(nums)
    counter = [0] * (m + 1)
    for num in nums:
        counter[num] += 1
    
    i = 0
    for num in range(m + 1):
        for _ in range(counter[num]):
            nums[i] = num
            i += 1
```

### 完整实现（稳定排序）

```python
def counting_sort(nums: list[int]):
    """ 计数排序（完整实现，可排序对象，稳定）"""
    m = max(nums)
    counter = [0] * (m + 1)
    for num in nums:
        counter[num] += 1
    
    # 求前缀和，将"出现次数"转换为"尾索引"
    for i in range(m):
        counter[i + 1] += counter[i]
    
    # 倒序遍历，将各元素填入结果数组
    n = len(nums)
    res = [0] * n
    for i in range(n - 1, -1, -1):
        num = nums[i]
        res[counter[num] - 1] = num
        counter[num] -= 1
    
    for i in range(n):
        nums[i] = res[i]
```

> **为什么倒序遍历？** 倒序遍历可以保持相同元素的相对顺序，实现稳定排序。

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(n + m)，通常趋向 O(n) |
| 空间复杂度 | O(n + m)，非原地排序 |
| 稳定性 | ✅ 稳定排序 |

### 局限性

- 仅适用于非负整数
- 当 n ≪ m 时，时间复杂度可能比 O(n log n) 算法还慢

## 11.10 基数排序（Radix Sort）

**基数排序**利用数字各位之间的递进关系，依次对每一位进行排序，从而得到最终的排序结果。

```python
def digit(num: int, exp: int) -> int:
    """ 获取元素 num 的第 k 位，exp = 10^(k-1)"""
    return (num // exp) % 10

def counting_sort_digit(nums: list[int], exp: int):
    """ 计数排序（根据 nums 第 k 位排序）"""
    counter = [0] * 10
    n = len(nums)
    for i in range(n):
        d = digit(nums[i], exp)
        counter[d] += 1
    for i in range(1, 10):
        counter[i] += counter[i - 1]
    res = [0] * n
    for i in range(n - 1, -1, -1):
        d = digit(nums[i], exp)
        j = counter[d] - 1
        res[j] = nums[i]
        counter[d] -= 1
    for i in range(n):
        nums[i] = res[i]

def radix_sort(nums: list[int]):
    """ 基数排序"""
    m = max(nums)
    exp = 1
    while exp <= m:
        counting_sort_digit(nums, exp)
        exp *= 10
```

> **为什么从最低位开始排序？** 后一轮排序会覆盖前一轮的结果，高位优先级高于低位，因此先排低位。

### 算法特性

| 特性 | 值 |
|------|-----|
| 时间复杂度 | O(nk)，通常趋向 O(n) |
| 空间复杂度 | O(n + d)，非原地排序 |
| 稳定性 | ✅ 稳定排序（当计数排序稳定时） |

### Q&A 精选

**Q：排序算法稳定性在什么情况下是必需的？**

A：多级排序场景。例如学生有姓名和身高两个属性，先按姓名排序，再按身高排序。如果排序算法不稳定，姓名的有序性可能被破坏。

**Q：哨兵划分为什么要先"从右往左查找"？**

A：以最左端元素为基准数时，必须先从右往左查找。否则最后一步交换可能将比基准数更大的元素交换至数组最左端，导致划分失败。

**Q：快速排序的递归深度优化为什么选短的数组递归？**

A：较短子数组的长度不会超过 n/2，因此递归深度不超过 log n。

**Q：数组所有元素都相等时，快速排序时间复杂度是 O(n²) 吗？**

A：是的。可以将数组划分为三部分：小于、等于、大于基准数。仅递归小于和大于两部分，一轮划分即可完成排序。

## 本章小结

1. **冒泡排序**：通过交换相邻元素，可通过标志位优化至 O(n)
2. **插入排序**：单元操作少，小数据量时最快
3. **快速排序**：哨兵划分 + 分治，平均 O(n log n)，应用最广泛
4. **归并排序**：稳定 O(n log n)，链表排序可优化至 O(1)
5. **堆排序**：原地 O(n log n)，适合实现优先级队列
6. **桶排序**：分治策略，适合大体量数据
7. **计数排序**：桶排序特例，适合数据量大但范围小
8. **基数排序**：逐位排序，要求数据可表示为固定位数

> 没有一种排序算法能同时满足所有优点，实际应用中需根据数据特性选择合适的算法。
