## 2.1 算法效率评估

### 两个层面的目标
1. **找到问题解法**：算法需要在规定的输入范围内可靠地求得问题的正确解
2. **寻求最优解法**：同一个问题可能存在多种解法，我们希望找到尽可能高效的算法

### 效率的两个维度
- **时间效率**：算法运行速度的快慢
- **空间效率**：算法占用内存空间的大小

简而言之，我们的目标是设计"既快又省"的数据结构与算法。

### 效率评估方法

#### 方法一：实际测试
```python
# 直接运行算法并监控时间和内存
import time

start = time.time()
# 算法代码
end = time.time()
print(f"运行时间: {end - start}")
```

**局限性**：
- 难以排除测试环境的干扰因素（硬件配置影响）
- 展开完整测试非常耗费资源

#### 方法二：理论估算（复杂度分析）

复杂度分析能够体现算法运行所需的时间和空间资源与输入数据大小之间的关系。它描述了随着输入数据大小的增加，算法执行所需时间和空间的增长趋势。

**优势**：
1. 独立于测试环境，分析结果适用于所有运行平台
2. 可以体现不同数据量下的算法效率，尤其是在大数据量下的算法性能

## 2.2 迭代与递归

### 迭代（Iteration）

迭代是通过循环结构重复执行代码块的过程。

#### for循环
```python
def for_loop(n: int) -> int:
    """for循环迭代"""
    result = 0
    for i in range(n):
        result += i
    return result

# 示例：计算 0+1+2+...+(n-1)
# 时间复杂度：O(n)
```

#### while循环
```python
def while_loop(n: int) -> int:
    """while循环迭代"""
    result = 0
    i = 0
    while i < n:
        result += i
        i += 1
    return result

# 示例：计算 0+1+2+...+(n-1)
# 时间复杂度：O(n)
```

#### 嵌套循环
```python
def nested_loop(n: int) -> int:
    """嵌套循环"""
    result = 0
    for i in range(n):
        for j in range(n):
            result += i * j
    return result

# 时间复杂度：O(n²)
```

### 递归（Recursion）

递归是函数调用自身的过程，需要设置终止条件防止无限循环。

```python
def recur(n: int) -> int:
    """递归函数"""
    # 终止条件
    if n == 1:
        return 1
    # 递归调用
    return n + recur(n - 1)

# 示例：计算 1+2+3+...+n
# 时间复杂度：O(n)
# 空间复杂度：O(n)（调用栈深度）
```

### 递归树

递归调用可以可视化为递归树：
```
recur(5)
├── recur(4)
│   ├── recur(3)
│   │   ├── recur(2)
│   │   │   ├── recur(1)
│   │   │   └── return 1
│   │   └── return 3
│   └── return 6
├── return 10
```

### 迭代 vs 递归对比

| 维度 | 迭代 | 递归 |
|------|------|------|
| 实现方式 | 循环结构 | 函数自调用 |
| 时间复杂度 | O(n)（单层） | O(n) |
| 空间复杂度 | O(1) | O(n)（调用栈） |
| 代码可读性 | 较直观 | 较简洁 |
| 适用场景 | 简单循环操作 | 分治、树结构 |

## 2.3 时间复杂度

### 什么是时间复杂度

时间复杂度是衡量算法运行时间随输入数据规模增长趋势的指标。它不关注具体的运行时间，而是关注增长趋势。

### 大O表示法（Big O Notation）

大O表示法描述的是算法的渐近上界，即最坏情况下算法执行时间的增长级别。

**数学定义**：若存在正实数 c 和实数 n₀，使得对于所有 n > n₀，均有 T(n) ≤ c·f(n)，则记为 T(n) = O(f(n))。

```
T(n) = 3 + 2n
     ↓ 简化
T(n) = O(n)
```

### 推算方法

#### 第一步：统计操作数量

给定算法：
```python
def algorithm(n: int) -> int:
    a = 1          # +1（常数）
    a = a + 1      # +1（常数）
    a = a * 2      # +1（常数）
    
    for i in range(n):  # 循环 n 次
        print(i)        # +1
        for j in range(n):  # 内层循环 n 次
            print(i, j)     # +1
    
    return a
```

操作数量：T(n) = 3 + 2n + n² = n² + 2n + 3

#### 第二步：判断渐近上界

简化原则：
1. **忽略常数项**（3 → 0）
2. **省略所有系数**（2n → n）
3. **循环嵌套时使用乘法**

最终：T(n) = O(n²)

### 常见时间复杂度类型

| 复杂度 | 函数 | 名称 | 示例 |
|--------|------|------|------|
| O(1) | 常数 | 常数阶 | 数组随机访问 |
| O(log n) | 对数 | 对数阶 | 二分查找 |
| O(n) | 线性 | 线性阶 | 数组遍历 |
| O(n log n) | 线性对数 | 线性对数阶 | 快速排序 |
| O(n²) | 平方 | 平方阶 | 冒泡排序 |
| O(n³) | 立方 | 立方阶 | 矩阵乘法 |
| O(2ⁿ) | 指数 | 指数阶 | 斐波那契递归 |
| O(n!) | 阶乘 | 阶乘阶 | 全排列 |

### 复杂度对比图

```

O(log n)    ███████████████████████████
O(n)        ██████████████████████████
O(n log n)  ████████████████████████████

O(2ⁿ)       ██████████████████████████████████████████████████████████████

增长趋势：常数阶 < 对数阶 < 线性阶 < 线性对数阶 < 平方阶 < 立方阶 < 指数阶 < 阶乘阶
```

### 各复杂度详解

#### 1. 常数阶 O(1)
```python
def constant(n: int) -> int:
    """常数阶时间复杂度"""
    size = 100000
    for _ in range(size):
        print(0)
    # 无论n多大，操作数量固定 → O(1)
    return 1

# 特点：操作数量与输入规模无关
# 应用：数组随机访问、哈希表查找
```

#### 2. 对数阶 O(log n)
```python
def logarithmic(n: int) -> int:
    """对数阶时间复杂度"""
    count = 0
    i = 1
    while i < n:
        i *= 2  # 每次规模减半
        count += 1
    # 循环次数: log₂n → O(log n)
    return count

# 数学原理：2^m = n ⟹ m = log₂n
# 应用：二分查找、平衡二叉树
```

#### 3. 线性阶 O(n)
```python
def linear(n: int) -> int:
    """线性阶时间复杂度"""
    total = 0
    for i in range(n):
        total += i
    # 总操作数: n → O(n)
    return total

# 应用：数组遍历、链表遍历
```

#### 4. 线性对数阶 O(n log n)
```python
def linear_logarithmic(n: int) -> int:
    """线性对数阶时间复杂度"""
    count = 0
    for i in range(n):          # O(n)
        j = 1
        while j < n:            # O(log n)
            j *= 2
            count += 1
    # 总复杂度: n × log n → O(n log n)
    return count

# 应用：归并排序、快速排序
```

#### 5. 平方阶 O(n²)
```python
def quadratic(n: int) -> int:
    """平方阶时间复杂度"""
    count = 0
    for i in range(n):          # 外层循环n次
        for j in range(n):      # 内层循环n次
            count += 1
    # 总操作数: n² → O(n²)
    return count
```

#### 6. 指数阶 O(2ⁿ)
```python
def exponential(n: int) -> int:
    """指数阶时间复杂度 - 斐波那契数列"""
    if n <= 1:
        return n
    return exponential(n - 1) + exponential(n - 2)

# 问题：每次调用产生两个子调用，递归树呈指数增长
# 优化：使用动态规划或记忆化搜索 → O(n)
```

#### 7. 阶乘阶 O(n!)
```python
def factorial(n: int) -> int:
    """阶乘阶时间复杂度 - 全排列"""
    if n == 0:
        return 1
    result = 0
    for i in range(n):
        result += factorial(n - 1)
    return result

# 应用：旅行商问题、全排列生成
```

### 最佳、最差、平均时间复杂度

```python
def find_target(nums: list, target: int) -> int:
    """查找目标元素"""
    for i, num in enumerate(nums):
        if num == target:
            return i  # 找到目标
    return -1  # 未找到

# 最佳情况：O(1) - 目标在第一个位置
# 最坏情况：O(n) - 目标在最后或不存在
# 平均情况：O(n) - 平均查找 n/2 次
```

## 2.4 空间复杂度

### 什么是空间复杂度

空间复杂度用于衡量算法占用内存空间随输入数据规模增长的趋势。

### 常见空间复杂度类型

| 复杂度 | 名称 | 说明 |
|--------|------|------|
| O(1) | 常数阶 | 占用空间固定 |
| O(n) | 线性阶 | 占用空间与n成正比 |
| O(n²) | 平方阶 | 占用空间与n²成正比 |

### 空间复杂度示例

#### O(1) - 常数阶
```python
def constant_space(n: int) -> int:
    """空间复杂度 O(1)"""
    a = 1
    b = 2
    c = a + b
    # 只使用常数个变量
    return c
```

#### O(n) - 线性阶
```python
def linear_space(n: int) -> list:
    """空间复杂度 O(n)"""
    nums = []
    for i in range(n):
        nums.append(i)
    # 占用 n 个元素的数组空间
    return nums
```

#### O(n²) - 平方阶
```python
def matrix_space(n: int) -> list:
    """空间复杂度 O(n²)"""
    matrix = []
    for i in range(n):
        row = []
        for j in range(n):
            row.append(i * j)
        matrix.append(row)
    # 占用 n×n 的二维数组空间
    return matrix
```

### 递归调用栈空间

```python
def recur_stack(n: int) -> int:
    """递归调用占用栈空间"""
    if n == 1:
        return 1
    return n * recur_stack(n - 1)

# 调用过程：
# recur_stack(5) → recur_stack(4) → ... → recur_stack(1)
# 调用栈深度为 n，占用空间 O(n)
```

### 迭代 vs 递归空间复杂度

| 算法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 迭代 | O(n) | O(1) |
| 递归 | O(n) | O(n) |

```python
# 迭代版本 - O(1) 空间
def sum_iterative(n: int) -> int:
    result = 0
    for i in range(n):
        result += i
    return result

# 递归版本 - O(n) 空间（调用栈）
def sum_recursive(n: int) -> int:
    if n <= 1:
        return n
    return n + sum_recursive(n - 1)
```

## 2.5 小结

### 本章要点

1. **算法效率评估**
   - 时间效率：运行速度的快慢
   - 空间效率：占用内存的大小
   - 目标：设计"既快又省"的算法

2. **复杂度分析方法**
   - 实际测试：受环境和资源限制
   - 理论估算（推荐）：独立于平台，反映增长趋势

3. **时间复杂度**
   - 大O表示法描述渐近上界
   - 推算方法：统计操作数量 → 判断渐近上界
   - 常见复杂度：O(1) < O(log n) < O(n) < O(n log n) < O(n²)

4. **空间复杂度**
   - 包括算法本身占用和运行时的额外占用
   - 递归调用栈空间
   - 可以用空间换时间

### 复杂度分析实战技巧

```python
# 技巧1：识别循环结构
def trick1(n: int) -> None:
    for i in range(n):      # O(n)
        for j in range(n):  # O(n)
            print(i, j)     # 总: O(n²)

# 技巧2：双指针/滑动窗口
def trick2(nums: list) -> int:
    left, right = 0, len(nums) - 1
    while left < right:
        # 操作
        left += 1
        right -= 1
    # 总: O(n)

# 技巧3：分治策略
def trick3(n: int) -> int:
    if n <= 1:
        return n
    mid = n // 2
    left = trick3(mid)      # O(log n)
    right = trick3(mid)     # O(log n)
    return left + right      # 总: O(n log n)
```
