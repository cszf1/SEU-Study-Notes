## 4.1 数组

### 4.1.1 数组基础

**数组（Array）** 是一种线性数据结构，将相同类型的元素存储在**连续的内存空间**中。元素在数组中的位置称为**索引（Index）**。

```python
# 初始化数组
arr: list[int] = [0] * 5       # [0, 0, 0, 0, 0] - 无初始值
nums: list[int] = [1, 3, 2, 5, 4]  # 有初始值
```

### 4.1.2 访问元素

数组元素被存储在连续的内存空间中，给定数组首地址和索引，可以直接计算元素内存地址。

```python
import random

def random_access(nums: list[int]) -> int:
    """随机访问元素"""
    random_index = random.randint(0, len(nums) - 1)
    random_num = nums[random_index]
    return random_num
```

> **核心特性**：数组可以在 **O(1)** 时间内随机访问任意元素。

### 4.1.3 插入元素

在数组中间插入元素需要将该位置之后的所有元素向后移动一位：

```python
def insert(nums: list[int], index: int, value: int):
    """在索引 index 处插入元素"""
    # 将 index 之后的所有元素向后移动一位
    for i in range(len(nums) - 1, index, -1):
        nums[i] = nums[i - 1]
    nums[index] = value
```

> **时间复杂度**：O(n)，因为需要移动元素。

### 4.1.4 删除元素

删除数组中索引 i 处的元素，需要将 i 之后的所有元素向前移动一位：

```python
def remove(nums: list[int], index: int):
    """删除索引 index 处的元素"""
    for i in range(index, len(nums) - 1):
        nums[i] = nums[i + 1]
```

### 4.1.5 遍历数组

```python
def traverse(nums: list[int]):
    """遍历数组"""
    count = 0
    
    # 通过索引遍历
    for i in range(len(nums)):
        count += nums[i]
    
    # 直接遍历元素
    for num in nums:
        count += num
    
    # 同时遍历索引和元素
    for i, num in enumerate(nums):
        count += nums[i]
        count += num
```

### 4.1.6 查找元素

线性查找：遍历数组判断元素值是否匹配：

```python
def find(nums: list[int], target: int) -> int:
    """在数组中查找指定元素"""
    for i in range(len(nums)):
        if nums[i] == target:
            return i
    return -1
```

> **时间复杂度**：O(n)，称为**线性查找**。

### 4.1.7 扩容数组

数组长度固定，扩容需要重新建立更大的数组并复制元素：

```python
def extend(nums: list[int], enlarge: int) -> list[int]:
    """扩展数组长度"""
    res = [0] * (len(nums) + enlarge)
    for i in range(len(nums)):
        res[i] = nums[i]
    return res
```

> **时间复杂度**：O(n)，大数据量时开销很大。

### 4.1.8 数组的优点与局限性

| 优点 | 局限性 |
|------|--------|
| 空间效率高：无额外结构开销 | 插入与删除效率低：需移动大量元素 |
| 支持随机访问：O(1) 时间 | 长度不可变：扩容开销大 |
| 缓存局部性好：访问效率高 | 空间浪费：可能分配过多空间 |

### 4.1.9 数组典型应用

- **随机访问**：随机抽样、根据索引随机抽样
- **排序和搜索**：快排、归并排序、二分查找
- **查找表**：字符到 ASCII 码的映射
- **机器学习**：向量、矩阵、张量运算
- **数据结构实现**：栈、队列、哈希表、堆、图的实现

## 4.2 链表

### 4.2.1 链表基础

**链表（Linked List）** 是一种线性数据结构，各节点通过"引用"连接，节点可以分散存储在内存各处，无须连续空间。

```python
class ListNode:
    """链表节点类"""
    def __init__(self, val: int):
        self.val: int = val      # 节点值
        self.next: ListNode | None = None  # 指向下一节点的引用
```

### 4.2.2 初始化链表

```python
# 初始化链表 1 -> 3 -> 2 -> 5 -> 4
n0 = ListNode(1)
n1 = ListNode(3)
n2 = ListNode(2)
n3 = ListNode(5)
n4 = ListNode(4)

# 构建节点之间的引用
n0.next = n1
n1.next = n2
n2.next = n3
n3.next = n4
```

### 4.2.3 插入节点

在相邻节点之间插入新节点只需改变两个引用，时间复杂度 O(1)：

```python
def insert(n0: ListNode, P: ListNode):
    """在节点 n0 之后插入节点 P"""
    n1 = n0.next
    P.next = n1
    n0.next = P
```

### 4.2.4 删除节点

删除节点只需改变一个引用，时间复杂度 O(1)：

```python
def remove(n0: ListNode):
    """删除链表 n0 之后的首个节点"""
    if not n0.next:
        return
    P = n0.next
    n1 = P.next
    n0.next = n1
```

### 4.2.5 访问节点

链表不支持随机访问，访问第 i 个节点需要循环 i-1 次：

```python
def access(head: ListNode, index: int) -> ListNode | None:
    """访问链表中索引为 index 的节点"""
    for _ in range(index):
        if not head:
            return None
        head = head.next
    return head
```

> **时间复杂度**：O(n)

### 4.2.6 查找节点

```python
def find(head: ListNode, target: int) -> int:
    """在链表中查找值为 target 的首个节点"""
    index = 0
    while head:
        if head.val == target:
            return index
        head = head.next
        index += 1
    return -1
```

### 4.2.7 数组 vs 链表

| | 数组 | 链表 |
|---|---|---|
| 存储方式 | 连续内存空间 | 分散内存空间 |
| 容量扩展 | 长度不可变 | 可灵活扩展 |
| 内存效率 | 元素占用少、但可能浪费 | 元素占用多（需存指针） |
| 访问元素 | O(1) | O(n) |
| 添加元素 | O(n) | O(1) |
| 删除元素 | O(n) | O(1) |

### 4.2.8 常见链表类型

**1. 单向链表**：普通链表，首到尾指向 None

**2. 环形链表**：尾节点指向头节点，可任意节点作为头节点

**3. 双向链表**：同时记录前后两个方向的引用

```python
class ListNode:
    """双向链表节点类"""
    def __init__(self, val: int):
        self.val: int = val           # 节点值
        self.next: ListNode | None = None  # 指向后继节点
        self.prev: ListNode | None = None   # 指向前驱节点
```

### 4.2.9 链表典型应用

| 链表类型 | 典型应用 |
|----------|----------|
| 单向链表 | 栈、队列、哈希表（链式地址）、图的邻接表 |
| 双向链表 | 红黑树/B树（访问父节点）、浏览器历史、LRU算法 |
| 环形链表 | 时间片轮转调度、数据缓冲区（音视频播放） |

## 4.3 列表

### 4.3.1 列表基础

**列表（List）** 是抽象数据结构，支持元素访问、修改、添加、删除、遍历，无须考虑容量限制。

- 列表可以基于链表或数组实现
- 大多数编程语言的列表基于**动态数组**实现（Python list、Java ArrayList、C++ vector）

### 4.3.2 列表常用操作

```python
# 初始化
nums1: list[int] = []              # 无初始值
nums: list[int] = [1, 3, 2, 5, 4]  # 有初始值

# 访问元素
num: int = nums[1]     # O(1) 访问

# 更新元素
nums[1] = 0

# 尾部添加元素 - O(1)
nums.append(6)

# 插入元素 - O(n)
nums.insert(2, 8)

# 删除元素 - O(n)
nums.pop(2)

# 遍历
for i in range(len(nums)):
    print(nums[i])
for num in nums:
    print(num)

# 拼接列表
nums1 = [6, 8, 7, 10, 9]
nums += nums1

# 排序
nums.sort()
```

### 4.3.3 列表实现

简易版列表实现：

```python
class MyList:
    """列表类"""
    def __init__(self):
        self._capacity: int = 10          # 列表容量
        self._arr: list[int] = [0] * self._capacity  # 底层数组
        self._size: int = 0               # 当前元素数量
        self._extend_ratio: int = 2       # 扩容倍数
    
    def size(self) -> int:
        """获取列表长度"""
        return self._size
    
    def capacity(self) -> int:
        """获取列表容量"""
        return self._capacity
    
    def get(self, index: int) -> int:
        """访问元素"""
        if index < 0 or index >= self._size:
            raise IndexError("索引越界")
        return self._arr[index]
    
    def add(self, num: int):
        """在尾部添加元素"""
        if self._size == self._capacity:
            self._extend_capacity()
        self._arr[self._size] = num
        self._size += 1
    
    def insert(self, num: int, index: int):
        """在中间插入元素"""
        if index < 0 or index >= self._size:
            raise IndexError("索引越界")
        if self._size == self._capacity:
            self._extend_capacity()
        # 元素向后移动
        for j in range(self._size - 1, index - 1, -1):
            self._arr[j + 1] = self._arr[j]
        self._arr[index] = num
        self._size += 1
    
    def remove(self, index: int) -> int:
        """删除元素"""
        if index < 0 or index >= self._size:
            raise IndexError("索引越界")
        num = self._arr[index]
        for j in range(index, self._size - 1):
            self._arr[j] = self._arr[j + 1]
        self._size -= 1
        return num
    
    def _extend_capacity(self):
        """列表扩容"""
        self._arr = self._arr + [0] * self._capacity * (self._extend_ratio - 1)
        self._capacity = len(self._arr)
```

## 4.4 内存与缓存 *

### 4.4.1 计算机存储设备

| | 硬盘 | 内存 | 缓存 |
|---|---|---|---|
| 用途 | 长期存储数据 | 临时存储运行程序 | 存储频繁访问的数据 |
| 易失性 | 断电不丢失 | 断电丢失 | 断电丢失 |
| 容量 | TB级 | GB级 | MB级 |
| 速度 | 几百~几千 MB/s | 几十 GB/s | 几十~几百 GB/s |
| 价格 | 几毛~几元/GB | 几十~几百元/GB | 随CPU计价 |

**存储层次**：硬盘 → 内存 → 缓存（越上层越快、越小、越贵）

### 4.4.2 内存效率对比

| | 数组 | 链表 |
|---|---|---|
| 空间效率 | 高（元素紧密排列） | 低（需存储指针） |
| 扩容成本 | 高（需复制全部元素） | 低（动态分配） |
| 碎片化 | 不易导致碎片化 | 易导致内存碎片化 |

### 4.4.3 缓存效率

**缓存加载机制**：
- **缓存行**：以行为单位加载，而非单字节
- **预取机制**：预测访问模式，提前加载数据
- **空间局部性**：加载某数据时，也加载附近数据
- **时间局部性**：保留最近访问的数据

**数组缓存效率更高的原因**：
1. 元素紧凑，有效数据更多
2. 数据访问模式更具可预测性
3. 空间局部性好，附近数据更可能被访问

> **结论**：数组通常比链表性能更好，但应根据具体场景选择。

## 4.5 小结

### 重点回顾

1. **数组 vs 链表**：两种基本数据结构，代表连续存储和分散存储
   - 数组：随机访问 O(1)，插入删除 O(n)，长度固定
   - 链表：插入删除 O(1)，访问 O(n)，长度灵活

2. **常见链表类型**：单向链表、环形链表、双向链表

3. **列表**：基于动态数组实现，保留数组优势同时可灵活调整长度

4. **缓存重要性**：数组缓存命中率高，通常比链表更高效

### Q&A 精选

**Q: 为什么数组要求相同类型元素？**
A: 数组通过偏移量计算地址 `地址 = 首地址 + 元素长度 × 索引`，不同类型长度不同无法计算。

**Q: 链表插入删除 O(1)，为什么查找还是 O(n)？**
A: 如果已找到节点位置，增删确实 O(1)。但通常需要先查找，所以总复杂度是 O(n)。

**Q: 为什么 C++ STL 的 std::list 不常用？**
A: 空间开销大（双向指针）、缓存不友好（数据不连续）。

**Q: Python 中 `[[0]] * n` 创建的二维列表有何问题？**
A: 所有 `[0]` 是同一对象引用，修改一个会影响所有。应用 `[[0] for _ in range(n)]`。
