# 第12章 分治 (Divide and Conquer)

## 12.1 分治算法

### 核心概念

**分治（Divide and Conquer）**：将原问题分解为规模更小的子问题，分别求解后再合并得到原问题的解。

### 两个阶段

| 阶段 | 说明 |
|------|------|
| **分（划分）** | 递归地将原问题分解为子问题，直至达到最小子问题 |
| **治（合并）** | 从已知解的最小子问题开始，从底至顶合并解 |

### 分治问题判断三要素

1. **问题可分解**：原问题可分解成规模更小、类似的子问题
2. **子问题独立**：子问题之间没有重叠，互不依赖，可独立解决
3. **解可合并**：原问题的解可通过合并子问题的解得来

### 分治提升效率的原因

#### 1. 操作数量优化

以冒泡排序为例，划分后时间复杂度从 $O(n^2)$ 变为 $O(\frac{n^2}{2}+2n)$，当 $n > 4$ 时效率提升。

不断细分直至子数组只有一个元素 → 归并排序，时间复杂度 $O(n \log n)$

#### 2. 并行计算优化

分治生成的子问题相互独立，可并行解决，在多核环境中显著提升效率。

### 分治典型应用

**经典算法问题：**
- 寻找最近点对
- 大整数乘法（Karatsuba算法）
- 矩阵乘法（Strassen算法）
- 汉诺塔问题
- 求解逆序对

**数据结构与算法：**
- 二分查找
- 归并排序、快速排序、堆排序
- 桶排序
- 二叉搜索树、AVL树、红黑树、B树、B+树
- 堆的各种操作

## 12.2 分治搜索策略

### 分治视角看二分查找

二分查找每轮将问题分解为子问题（在数组的一半中搜索），直至找到目标或区间为空。

```python
from typing import List

def dfs(nums: List[int], target: int, i: int, j: int) -> int:
    """二分查找：问题f(i,j)"""
    # 若区间为空，代表无目标元素，则返回-1
    if i > j:
        return -1
    
    # 计算中点索引m
    m = (i + j) // 2
    
    if nums[m] < target:
        # 递归子问题f(m+1, j)
        return dfs(nums, target, m + 1, j)
    elif nums[m] > target:
        # 递归子问题f(i, m-1)
        return dfs(nums, target, i, m - 1)
    else:
        # 找到目标元素，返回其索引
        return m

def binary_search(nums: List[int], target: int) -> int:
    """二分查找"""
    n = len(nums)
    # 求解问题f(0, n-1)
    return dfs(nums, target, 0, n - 1)
```

### 分治搜索 vs 暴力搜索

- 暴力搜索每轮只能排除 **1个** 选项
- 分治搜索每轮可以排除 **一半** 选项

## 12.3 构建二叉树问题

### 问题描述

给定前序遍历 `preorder` 和中序遍历 `inorder`，构建二叉树。

```python
# Definition for a binary tree node
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def dfs(preorder: List[int], inorder_map: dict, i: int, l: int, r: int) -> TreeNode:
    """构建二叉树：分治"""
    # 子树区间为空时终止
    if r - l < 0:
        return None
    
    # 初始化根节点
    root = TreeNode(preorder[i])
    
    # 查询m，从而划分左右子树
    m = inorder_map[preorder[i]]
    
    # 子问题：构建左子树
    root.left = dfs(preorder, inorder_map, i + 1, l, m - 1)
    # 子问题：构建右子树
    root.right = dfs(preorder, inorder_map, i + 1 + (m - l), m + 1, r)
    
    return root

def build_tree(preorder: List[int], inorder: List[int]) -> TreeNode:
    """构建二叉树"""
    # 初始化哈希表，存储inorder元素到索引的映射
    inorder_map = {val: i for i, val in enumerate(inorder)}
    return dfs(preorder, inorder_map, 0, 0, len(inorder) - 1)
```

### 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | $O(n)$ | 每个节点初始化使用 $O(1)$ 时间 |
| 空间复杂度 | $O(n)$ | 哈希表存储 $O(n)$，递归栈最深 $O(n)$ |

## 12.4 汉诺塔问题

### 问题描述

将n个圆盘从柱子A移动到柱子C，保持顺序不变，规则：
1. 每次只能移动一个圆盘
2. 小圆盘必须在大圆盘之上

### 分治策略

将问题 $f(n)$ 分解为两个子问题 $f(n-1)$ 和一个子问题 $f(1)$：

1. 将 $n-1$ 个圆盘借助C从A移至B
2. 将剩余1个圆盘从A直接移至C
3. 将 $n-1$ 个圆盘借助A从B移至C

```python
def move(src: list, tar: list):
    """移动一个圆盘"""
    # 从src顶部拿出一个圆盘
    pan = src.pop()
    # 将圆盘放入tar顶部
    tar.append(pan)

def dfs(i: int, src: list, buf: list, tar: list):
    """解汉诺塔问题f(i)"""
    # 若src只剩下一个圆盘，则直接将其移到tar
    if i == 1:
        move(src, tar)
        return
    
    # 子问题f(i-1)：将src顶部i-1个圆盘借助tar移到buf
    dfs(i - 1, src, tar, buf)
    
    # 子问题f(1)：将src剩余一个圆盘移到tar
    move(src, tar)
    
    # 子问题f(i-1)：将buf顶部i-1个圆盘借助src移到tar
    dfs(i - 1, buf, src, tar)

def solve_hanota(A: list, B: list, C: list):
    """求解汉诺塔问题"""
    n = len(A)
    # 将A顶部n个圆盘借助B移到C
    dfs(n, A, B, C)
```

### 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | $O(2^n)$ | 形成高度为n的递归树 |
| 空间复杂度 | $O(n)$ | 递归深度为n |

> **趣味知识**：传说中僧侣移动64个圆盘需要约5850亿年，远超宇宙年龄。

## 12.5 小结

### 重点回顾

1. **分治是重要的算法设计策略**，包括分（划分）和治（合并）两个阶段，通常基于递归实现

2. **分治问题三要素**：问题可分解、子问题独立、解可合并

3. **分治提升效率的原因**：
   - 减少操作数量
   - 有利于系统并行优化

4. **典型应用**：
   - 归并排序、快速排序（分治策略的排序算法）
   - 二分查找（分治搜索）
   - 构建二叉树、汉诺塔问题

### 复杂度对比

| 算法 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 归并排序（数组） | $O(n \log n)$ | $O(n)$ |
| 归并排序（链表） | $O(n \log n)$ | $O(1)$ |
| 构建二叉树 | $O(n)$ | $O(n)$ |
| 汉诺塔 | $O(2^n)$ | $O(n)$ |
