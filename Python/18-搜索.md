## 18.1 二分查找

**二分查找（Binary Search）** 是一种基于分治策略的高效搜索算法。它利用数据的有序性，每轮缩小一半搜索范围，直至找到目标元素或搜索区间为空为止。

### 问题描述

> 给定一个长度为 n 的数组 nums，元素按从小到大的顺序排列且不重复。请查找并返回元素 target 在该数组中的索引。若数组不包含该元素，则返回 -1。

### 算法步骤

1. 初始化指针 `i = 0` 和 `j = n - 1`，分别指向数组首元素和尾元素，代表搜索区间 `[0, n-1]`
2. 循环执行以下步骤：
   - 计算中点索引 `m = ⌊(i + j) / 2⌋`
   - 判断 `nums[m]` 和 `target` 的大小关系：
     - `nums[m] < target`：target 在区间 `[m+1, j]` 中，执行 `i = m + 1`
     - `nums[m] > target`：target 在区间 `[i, m-1]` 中，执行 `j = m - 1`
     - `nums[m] == target`：找到 target，返回索引 m
3. 若数组不包含目标元素，搜索区间最终会缩小为空，此时返回 -1

### 代码实现：双闭区间写法

```python
def binary_search(nums: list[int], target: int) -> int:
    """二分查找（双闭区间）"""
    # 初始化双闭区间 [0, n-1]，即 i, j 分别指向数组首元素、尾元素
    i, j = 0, len(nums) - 1
    
    # 循环，当搜索区间为空时跳出（当 i > j 时为空）
    while i <= j:
        m = (i + j) // 2  # 计算中点索引 m
        if nums[m] < target:
            i = m + 1  # 此情况说明 target 在区间 [m+1, j] 中
        elif nums[m] > target:
            j = m - 1  # 此情况说明 target 在区间 [i, m-1] 中
        else:
            return m  # 找到目标元素，返回其索引
    
    return -1  # 未找到目标元素，返回 -1
```

**复杂度分析**：
- 时间复杂度：O(log n) —— 区间每轮缩小一半，循环次数为 log₂n
- 空间复杂度：O(1) —— 指针 i 和 j 使用常数大小空间

### 18.1.1 区间表示方法

除了双闭区间，还有 **左闭右开** 区间 `[0, n)`，即左边界包含自身，右边界不包含自身。

```python
def binary_search_lcro(nums: list[int], target: int) -> int:
    """二分查找（左闭右开区间）"""
    # 初始化左闭右开区间 [0, n)
    i, j = 0, len(nums)
    
    # 循环，当搜索区间为空时跳出（当 i = j 时为空）
    while i < j:
        m = (i + j) // 2
        if nums[m] < target:
            i = m + 1
        elif nums[m] > target:
            j = m
        else:
            return m
    
    return -1
```

> 💡 **建议**：一般建议采用"双闭区间"的写法，因为左右边界操作对称，更不容易出错。

### 18.1.2 优点与局限性

**优点**：
- 时间效率高：在大数据量下，对数阶时间复杂度具有显著优势
- 无须额外空间：相较于需要借助额外空间的搜索算法（如哈希查找），更加节省空间

**局限性**：
- 仅适用于有序数据：若输入无序，排序的时间复杂度 O(n log n) 比线性查找更高
- 仅适用于数组：链表不支持跳跃式访问
- 小数据量下，线性查找反而更快

## 18.2 二分查找插入点

二分查找不仅可用于搜索目标元素，还可用于解决变种问题，比如搜索目标元素的插入位置。

### 18.2.1 无重复元素的情况

> 给定一个长度为 n 的有序数组 nums 和一个元素 target，数组不存在重复元素。现将 target 插入数组 nums 中，并保持其有序性。若数组中已存在元素 target，则插入到其左方。请返回插入后 target 在数组中的索引。

**关键发现**：二分结束时，`i` 指向首个大于 target 的元素，`j` 指向首个小于 target 的元素。因此当数组不包含 target 时，插入索引为 `i`。

```python
def binary_search_insertion_simple(nums: list[int], target: int) -> int:
    """二分查找插入点（无重复元素）"""
    i, j = 0, len(nums) - 1  # 初始化双闭区间 [0, n-1]
    
    while i <= j:
        m = (i + j) // 2
        if nums[m] < target:
            i = m + 1
        elif nums[m] > target:
            j = m - 1
        else:
            return m  # 找到 target，返回插入点 m
    
    return i  # 未找到 target，返回插入点 i
```

### 18.2.2 存在重复元素的情况

> 数组可能包含重复元素，将目标元素插入到最左边。

**核心思想**：当 `nums[m] == target` 时，仍然收缩右边界 `j = m - 1`，使指针 j 向小于 target 的元素靠近。

```python
def binary_search_insertion(nums: list[int], target: int) -> int:
    """二分查找插入点（存在重复元素）"""
    i, j = 0, len(nums) - 1
    
    while i <= j:
        m = (i + j) // 2
        if nums[m] < target:
            i = m + 1
        elif nums[m] > target:
            j = m - 1
        else:
            j = m - 1  # 首个小于 target 的元素在区间 [i, m-1] 中
    
    return i  # 返回插入点 i
```

## 18.3 二分查找边界

### 18.3.1 查找左边界

> 给定一个长度为 n 的有序数组 nums，其中可能包含重复元素。请返回数组中最左一个元素 target 的索引。若数组中不包含该元素，则返回 -1。

```python
def binary_search_left_edge(nums: list[int], target: int) -> int:
    """二分查找最左一个 target"""
    # 等价于查找 target 的插入点
    i = binary_search_insertion(nums, target)
    
    # 未找到 target，返回 -1
    if i == len(nums) or nums[i] != target:
        return -1
    
    return i
```

### 18.3.2 查找右边界

**方法一：复用查找左边界**

将查找最右一个 target 转化为查找最左一个 `target + 1`。

```python
def binary_search_right_edge(nums: list[int], target: int) -> int:
    """二分查找最右一个 target"""
    # 转化为查找最左一个 target + 1
    i = binary_search_insertion(nums, target + 1)
    
    # j 指向最右一个 target，i 指向首个大于 target 的元素
    j = i - 1
    
    # 未找到 target，返回 -1
    if j == -1 or nums[j] != target:
        return -1
    
    return j
```

**方法二：构造不存在元素**

- 查找最左 target → 转化为查找 `target - 0.5`
- 查找最右 target → 转化为查找 `target + 0.5`

## 18.4 哈希优化策略

### 问题描述

> 给定一个整数数组 nums 和一个目标元素 target，请在数组中搜索"和"为 target 的两个元素，并返回它们的数组索引。

### 18.4.1 暴力枚举：O(n²)

```python
def two_sum_brute_force(nums: list[int], target: int) -> list[int]:
    """方法一：暴力枚举"""
    # 两层循环，时间复杂度为 O(n²)
    for i in range(len(nums) - 1):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

### 18.4.2 哈希优化：O(n)

借助哈希表，键值对分别为数组元素和元素索引。

```python
def two_sum_hash_table(nums: list[int], target: int) -> list[int]:
    """方法二：辅助哈希表"""
    # 辅助哈希表，空间复杂度为 O(n)
    dic = {}
    
    # 单层循环，时间复杂度为 O(n)
    for i in range(len(nums)):
        if target - nums[i] in dic:
            return [dic[target - nums[i]], i]
        dic[nums[i]] = i
    
    return []
```

**优化效果**：将时间复杂度从 O(n²) 降至 O(n)，大幅提升运行效率。

## 18.5 重识搜索算法

搜索算法用于在数据结构中搜索满足特定条件的元素。

### 18.5.1 暴力搜索

| 算法 | 适用数据结构 | 时间复杂度 |
|------|-------------|-----------|
| 线性搜索 | 数组、链表 | O(n) |
| 广度优先搜索 (BFS) | 图、树 | O(n) |
| 深度优先搜索 (DFS) | 图、树 | O(n) |

### 18.5.2 自适应搜索

| 算法 | 适用数据结构 | 时间复杂度 | 预处理 |
|------|-------------|-----------|--------|
| 二分查找 | 数组 | O(log n) | 排序 O(n log n) |
| 哈希查找 | 通用 | O(1) | 建哈希表 O(n) |
| 树查找 | 二叉搜索树 | O(log n) | 建树 O(n log n) |

### 18.5.3 搜索方法选取

| 场景 | 推荐方法 |
|------|---------|
| 数据量小、频繁更新 | 线性搜索 |
| 大数据量、有序数据 | 二分查找 |
| 查询性能要求高、无需范围查询 | 哈希查找 |
| 海量数据、需要范围查询 | 树查找 |

## 18.6 小结

1. **二分查找**依赖数据的有序性，通过循环逐步缩减一半搜索区间来进行查找。要求输入数据有序，且仅适用于数组。

2. **暴力搜索**包括线性搜索（数组/链表）和 BFS/DFS（图/树），通用性好但时间复杂度 O(n) 较高。

3. **高效搜索方法**包括哈希查找、树查找和二分查找，时间复杂度可达 O(log n) 甚至 O(1)，但通常需要借助额外数据结构。

4. **哈希查找替换线性查找**是常用的优化策略，可将时间复杂度从 O(n) 降至 O(1)。

5. **实际选择**需要综合考虑数据规模、搜索性能要求、数据查询和更新频率等因素。
