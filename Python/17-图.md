## 17.1 图

**图（graph）** 是一种非线性数据结构，由**顶点（vertex）**和**边（edge）**组成。我们可以将图 G 抽象地表示为一组顶点 V 和一组边 E 的集合。

如果将顶点看作节点，将边看作连接各个节点的引用（指针），我们就可以将图看作一种从链表拓展而来的数据结构。**相较于线性关系（链表）和分治关系（树），网络关系（图）的自由度更高**，因而更为复杂。

### 17.1.1 图的常见类型与术语

#### 有向图 vs 无向图

根据边是否具有方向，可分为**无向图（undirected graph）**和**有向图（directed graph）**：
- 在无向图中，边表示两顶点之间的"双向"连接关系，例如微信或 QQ 中的"好友关系"
- 在有向图中，边具有方向性，即 A→B 和 A←B 两个方向的边是相互独立的，例如微博或抖音上的"关注"与"被关注"关系

#### 连通图 vs 非连通图

根据所有顶点是否连通，可分为**连通图（connected graph）**和**非连通图（disconnected graph）**：
- 对于连通图，从某个顶点出发，可以到达其余任意顶点
- 对于非连通图，从某个顶点出发，至少有一个顶点无法到达

#### 有权图 vs 无权图

还可以为边添加"权重"变量，从而得到**有权图（weighted graph）**。

#### 基本术语

- **邻接（adjacency）**：当两顶点之间存在边相连时，称这两顶点"邻接"
- **路径（path）**：从顶点 A 到顶点 B 经过的边构成的序列被称为从 A 到 B 的"路径"
- **度（degree）**：一个顶点拥有的边数
  - 对于有向图，**入度（in-degree）**表示有多少条边指向该顶点
  - **出度（out-degree）**表示有多少条边从该顶点指出

### 17.1.2 图的表示

#### 1. 邻接矩阵

设图的顶点数量为 n ，**邻接矩阵（adjacency matrix）**使用一个 n×n 大小的矩阵来表示图，每一行（列）代表一个顶点，矩阵元素代表边，用 1 或 0 表示两个顶点之间是否存在边。

```
邻接矩阵特点：
- 简单图中，顶点不能与自身相连，主对角线元素没有意义
- 对于无向图，两个方向的边等价，邻接矩阵关于主对角线对称
- 将邻接矩阵的元素从 1 和 0 替换为权重，则可表示有权图
```

**复杂度分析**：
- 时间复杂度：增删查改操作均为 O(1)
- 空间复杂度：O(n²)

#### 2. 邻接表

**邻接表（adjacency list）**使用 n 个链表来表示图，链表节点表示顶点。第 i 个链表对应顶点 i ，其中存储了该顶点的所有邻接顶点。

```
邻接表特点：
- 仅存储实际存在的边，总数通常远小于 n²，更加节省空间
- 需要通过遍历链表来查找边，时间效率不如邻接矩阵
- 可采用哈希表或红黑树优化查询效率
```

**复杂度分析**：
- 空间复杂度：O(|V| + |E|)

#### 对比总结

| 特性 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 空间复杂度 | O(n²) | O(\|V\| + \|E\|) |
| 增删边 | O(1) | O(1) |
| 查询边 | O(1) | O(n) |
| 适用场景 | 频繁查询边、顶点数量较小 | 存储稀疏图 |

> 💡 **核心理念**：邻接矩阵体现了"以空间换时间"，邻接表体现了"以时间换空间"。

### 17.1.3 图的常见应用

| 领域 | 顶点 | 边 | 图计算问题 |
|------|------|-----|-----------|
| 社交网络 | 用户 | 好友关系 | 潜在好友推荐 |
| 地铁线路 | 站点 | 站点间的连通性 | 最短路线推荐 |
| 太阳系 | 星体 | 星体间的万有引力作用 | 行星轨道计算 |

## 17.2 图基础操作

图的基础操作可分为对"边"的操作和对"顶点"的操作。

### 17.2.1 基于邻接矩阵的实现

```python
class GraphAdjMat:
    """基于邻接矩阵实现的无向图类"""

    def __init__(self, vertices: list[int], edges: list[list[int]]):
        """构造方法"""
        self.vertices: list[int] = []      # 顶点列表
        self.adj_mat: list[list[int]] = [] # 邻接矩阵
        
        # 添加顶点
        for val in vertices:
            self.add_vertex(val)
        # 添加边（edges 元素代表顶点索引）
        for e in edges:
            self.add_edge(e[0], e[1])

    def size(self) -> int:
        """获取顶点数量"""
        return len(self.vertices)

    def add_vertex(self, val: int):
        """添加顶点 O(n)"""
        n = self.size()
        self.vertices.append(val)
        # 在邻接矩阵中添加一行
        new_row = [0] * n
        self.adj_mat.append(new_row)
        # 在邻接矩阵中添加一列
        for row in self.adj_mat:
            row.append(0)

    def remove_vertex(self, index: int):
        """删除顶点 O(n²)"""
        if index >= self.size():
            raise IndexError()
        self.vertices.pop(index)
        self.adj_mat.pop(index)
        for row in self.adj_mat:
            row.pop(index)

    def add_edge(self, i: int, j: int):
        """添加边 O(1)"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError()
        # 无向图：邻接矩阵关于主对角线对称
        self.adj_mat[i][j] = 1
        self.adj_mat[j][i] = 1

    def remove_edge(self, i: int, j: int):
        """删除边 O(1)"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError()
        self.adj_mat[i][j] = 0
        self.adj_mat[j][i] = 0

    def print(self):
        """打印邻接矩阵"""
        print("顶点列表 =", self.vertices)
        print("邻接矩阵 =")
        for row in self.adj_mat:
            print("  ", row)
```

### 17.2.2 基于邻接表的实现

```python
from collections import defaultdict

class Vertex:
    """顶点类型"""
    def __init__(self, val: int):
        self.val = val
    
    def __hash__(self):
        return hash(self.val)
    
    def __eq__(self, other):
        return isinstance(other, Vertex) and self.val == other.val

class GraphAdjList:
    """基于邻接表实现的无向图类"""

    def __init__(self, vertices: list[int], edges: list[list[int]]):
        """构造方法"""
        self.adj_list: dict[Vertex, list[Vertex]] = defaultdict(list)
        
        # 添加顶点
        for val in vertices:
            self.add_vertex(val)
        # 添加边
        for e in edges:
            self.add_edge(e[0], e[1])

    def size(self) -> int:
        """获取顶点数量"""
        return len(self.adj_list)

    def add_vertex(self, val: int):
        """添加顶点"""
        if val not in [v.val for v in self.adj_list]:
            self.adj_list[Vertex(val)] = []

    def remove_vertex(self, index: int):
        """删除顶点 O(|V| + |E|)"""
        # 需要删除所有相关的边
        pass

    def add_edge(self, i: int, j: int):
        """添加边"""
        v1 = Vertex(i)
        v2 = Vertex(j)
        self.adj_list[v1].append(v2)
        self.adj_list[v2].append(v1)

    def remove_edge(self, i: int, j: int):
        """删除边"""
        v1 = Vertex(i)
        v2 = Vertex(j)
        if v2 in self.adj_list[v1]:
            self.adj_list[v1].remove(v2)
        if v1 in self.adj_list[v2]:
            self.adj_list[v2].remove(v1)

    def print(self):
        """打印邻接表"""
        print("邻接表 =")
        for vertex, adj_list in self.adj_list.items():
            adj_vals = [adj_v.val for adj_v in adj_list]
            print(f"  {vertex.val} -> {adj_vals}")
```

### 17.2.3 效率对比

| 操作 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 添加边 | O(1) | O(1) |
| 删除边 | O(1) | O(|E|) |
| 添加顶点 | O(n) | O(1) |
| 删除顶点 | O(n²) | O(|V| + |E|) |
| 查询边是否存在 | O(1) | O(deg) |
| 遍历顶点的邻接顶点 | O(n) | O(deg) |

## 17.3 图的遍历

树的遍历是图的遍历的一种特例。图遍历主要有两种策略：**广度优先遍历（BFS）** 和 **深度优先遍历（DFS）**。

### 17.3.1 广度优先遍历

**广度优先遍历是一种由近及远、层层扩张的搜索方式**，类似于水的波纹扩散，从起点出发，一层层向外扩展。

```python
from collections import deque

def graph_bfs(graph: GraphAdjList, start_vet: Vertex) -> list[Vertex]:
    """广度优先遍历"""
    # 顶点遍历序列
    res = []
    # 哈希集合，用于记录已被访问过的顶点
    visited = {start_vet}
    # 队列用于实现 BFS
    que = deque([start_vet])
    
    # 以顶点 vet 为起点，循环直至访问完所有顶点
    while que:
        vet = que.popleft()  # 队首顶点出队
        res.append(vet)      # 记录访问顶点
        
        # 遍历该顶点的所有邻接顶点
        for adj_vet in graph.adj_list[vet]:
            if adj_vet in visited:
                continue      # 跳过已被访问的顶点
            que.append(adj_vet)   # 只入队未访问的顶点
            visited.add(adj_vet)   # 标记该顶点已被访问
    
    return res
```

**复杂度分析**：
- 时间复杂度：O(|V| + |E|)
- 空间复杂度：O(|V|)

**特点**：
- 广度优先遍历只要求按"由近及远"的顺序遍历
- 多个相同距离的顶点的遍历顺序可以任意

### 17.3.2 深度优先遍历

**深度优先遍历是一种优先走到底、无路可走再回头的遍历方式**。通常基于递归实现。

```python
def graph_dfs(graph: GraphAdjList, start_vet: Vertex) -> list[Vertex]:
    """深度优先遍历"""
    res = []
    visited = set[Vertex]()
    
    def dfs(graph: GraphAdjList, visited: set, res: list, vet: Vertex):
        """深度优先遍历辅助函数"""
        res.append(vet)       # 记录访问顶点
        visited.add(vet)      # 标记该顶点已被访问
        
        # 遍历该顶点的所有邻接顶点
        for adj_vet in graph.adj_list[vet]:
            if adj_vet in visited:
                continue      # 跳过已被访问的顶点
            # 递归访问邻接顶点
            dfs(graph, visited, res, adj_vet)
    
    dfs(graph, visited, res, start_vet)
    return res
```

**复杂度分析**：
- 时间复杂度：O(|V| + |E|)
- 空间复杂度：O(|V|)（递归深度最大为 |V|）

**特点**：
- 先往哪个方向探索都可以，邻接顶点的顺序可以任意
- 与广度优先遍历类似，遍历序列的顺序也不是唯一的

### 17.3.3 BFS vs DFS 对比

| 特性 | BFS（广度优先） | DFS（深度优先） |
|------|----------------|-----------------|
| 核心思想 | 由近及远，层层扩张 | 走到底，无路可走再回头 |
| 数据结构 | 队列（Queue） | 栈/递归（Stack/Recursion） |
| 适用场景 | 最短路径、无权图最短路 | 连通分量、拓扑排序 |
| 空间复杂度 | O(n) | O(n) |

## 17.4 小结

### 重点回顾

1. **图的基本概念**：图由顶点和边组成，可以表示为一组顶点和一组边构成的集合。相比链表和树，图具有更高的自由度。

2. **图的分类**：
   - 有向图的边具有方向性
   - 连通图中的任意顶点均可达
   - 有权图的每条边都包含权重变量

3. **图的表示方法**：
   - **邻接矩阵**：增删查改效率高 O(1)，但空间复杂度高 O(n²)
   - **邻接表**：空间效率高 O(|V| + |E|)，但查询效率低
   - 体现了"以空间换时间"和"以时间换空间"的权衡

4. **图的遍历**：
   - **BFS**：由近及远、层层扩张，通常借助队列实现
   - **DFS**：优先走到底、无路可走时回溯，常基于递归实现

5. **树与图的关系**：树是图的一种特例，树的遍历也是图的遍历的一种特例。

### Q & A

**Q：路径的定义是顶点序列还是边序列？**

A：不同资料定义不同。在本文中，路径被视为一个**边序列**，因为两个顶点之间可能存在多条边连接。

**Q：非连通图中是否会有无法遍历到的点？**

A：是的。在非连通图中，从某个顶点出发，至少有一个顶点无法到达。遍历非连通图需要设置多个起点，以遍历到图的所有连通分量。

**Q：邻接表中顶点的顺序有要求吗？**

A：可以是任意顺序。但在实际应用中，可能需要按照指定规则排序，比如按顶点添加次序或顶点值大小，这样有助于快速查找"带有某种极值"的顶点。
