## 章节概述

动态规划(Dynamic Programming)是一种重要的算法范式，它将一个问题分解为一系列更小的子问题，并通过存储子问题的解来避免重复计算，从而大幅提升时间效率。

> 小溪汇入河流，江河汇入大海。动态规划将小问题的解汇集成大问题的答案，一步步引领我们走向解决问题的彼岸。

## 14.1 初探动态规划

### 问题引入：爬楼梯

**问题描述**：给定一个共有n阶的楼梯，你每步可以上1阶或者2阶，请问有多少种方案可以爬到楼顶？

**递推关系**：
$$dp[i] = dp[i-1] + dp[i-2]$$

其中 $dp[i]$ 表示爬到第 $i$ 阶的方案数。

### 方法一：暴力搜索（回溯）

```python
def climbing_stairs_backtrack(n: int) -> int:
    """爬楼梯：回溯（暴力搜索）"""
    choices = [1, 2]  # 可选择向上爬1阶或2阶
    state = []        # 从第0阶开始爬
    res = [0]         # 使用res[0]记录方案数量
    
    def backtrack(choices, state, n, res):
        # 如果超过楼梯顶部，剪枝
        if sum(state) > n:
            return
        # 如果到达楼梯顶部，记录方案
        if sum(state) == n:
            res[0] += 1
            return
        # 尝试每种选择
        for choice in choices:
            state.append(choice)
            backtrack(choices, state, n, res)
            state.pop()
    
    backtrack(choices, state, n, res)
    return res[0]
```

**时间复杂度**：$O(2^n)$ —— 指数阶，效率极低

### 方法二：记忆化搜索（备忘录）

从原问题(根节点)开始，递归地将较大子问题分解为较小子问题，直至解已知的最小子问题(叶节点)。

```python
from typing import List

def dfs(i: int, mem: List[int]) -> int:
    """记忆化搜索"""
    # 已知dp[1]和dp[2]，返回之
    if i == 1 or i == 2:
        return i
    # 若存在记录dp[i]，则直接返回
    if mem[i] != -1:
        return mem[i]
    # dp[i] = dp[i-1] + dp[i-2]
    count = dfs(i - 1, mem) + dfs(i - 2, mem)
    # 记录dp[i]
    mem[i] = count
    return count

def climbing_stairs_dfs_mem(n: int) -> int:
    """爬楼梯：记忆化搜索"""
    # mem[i]记录爬到第i阶的方案总数，-1代表无记录
    mem = [-1] * (n + 1)
    return dfs(n, mem)
```

**时间复杂度**：$O(n)$ —— 巨大飞跃！

**空间复杂度**：$O(n)$

### 方法三：动态规划（自底向上）

动态规划是一种"从底至顶"的方法：从最小子问题的解开始，迭代地构建更大子问题的解，直至得到原问题的解。

```python
def climbing_stairs_dp(n: int) -> int:
    """爬楼梯：动态规划"""
    if n == 1 or n == 2:
        return n
    # 初始化dp表，用于存储子问题的解
    dp = [0] * (n + 1)
    # 初始状态：预设最小子问题的解
    dp[1], dp[2] = 1, 2
    # 状态转移：从较小子问题逐步求解较大子问题
    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

**时间复杂度**：$O(n)$

**空间复杂度**：$O(n)$

### 空间优化（滚动变量）

由于 $dp[i]$ 只与 $dp[i-1]$ 和 $dp[i-2]$ 有关，无须用一个数组来存储所有子问题的解，只需两个变量滚动前进即可。

```python
def climbing_stairs_dp_comp(n: int) -> int:
    """爬楼梯：空间优化后的动态规划"""
    if n == 1 or n == 2:
        return n
    a, b = 1, 2  # a = dp[i-2], b = dp[i-1]
    for _ in range(3, n + 1):
        a, b = b, a + b  # 滚动更新
    return b
```

**空间复杂度**：$O(1)$

### 动态规划核心术语

| 术语 | 含义 |
|------|------|
| **状态** | 问题求解的特定阶段，如爬楼梯中的当前阶数 |
| **dp表** | 存储子问题解的数组或数据结构 |
| **初始状态** | 最小子问题对应的状态，如dp[1]、dp[2] |
| **状态转移方程** | 从子问题构建原问题的递推公式 |

## 14.2 动态规划问题特性

### 动态规划 vs 分治 vs 回溯

| 算法 | 子问题关系 | 特点 |
|------|-----------|------|
| **分治** | 相互独立 | 递归划分，合并子问题的解 |
| **动态规划** | 相互依赖（有重叠子问题） | 存储子问题的解避免重复计算 |
| **回溯** | 决策序列 | 尝试和回退穷举所有可能 |

### 两大特性

#### 1. 最优子结构

**定义**：原问题的最优解是从子问题的最优解构建来的。

**例题：爬楼梯最小代价**

```python
def min_cost_climbing_stairs_dp(cost: List[int]) -> int:
    """爬楼梯最小代价：动态规划"""
    n = len(cost)
    if n == 1 or n == 2:
        return cost[n]
    
    # 初始化dp表
    dp = [0] * (n + 1)
    # 初始状态
    dp[1], dp[2] = cost[1], cost[2]
    # 状态转移：dp[i] = min(dp[i-1], dp[i-2]) + cost[i]
    for i in range(3, n + 1):
        dp[i] = min(dp[i - 1], dp[i - 2]) + cost[i]
    return dp[n]
```

**空间优化版本**：
```python
def min_cost_climbing_stairs_dp_comp(cost: List[int]) -> int:
    """爬楼梯最小代价：空间优化"""
    n = len(cost)
    if n == 1 or n == 2:
        return cost[n]
    
    a, b = cost[1], cost[2]  # a = dp[i-2], b = dp[i-1]
    for i in range(3, n + 1):
        a, b = b, min(a, b) + cost[i]
    return b
```

#### 2. 无后效性

**定义**：给定一个确定的状态，它的发展只与当前状态有关，而与过去经历的所有状态无关。

**有后效性案例：带约束爬楼梯**

不能连续两轮跳1阶，此时状态需要扩展：

```python
def climbing_stairs_constraint_dp(n: int) -> int:
    """带约束爬楼梯：动态规划"""
    if n == 1 or n == 2:
        return 1
    
    # dp[i][j] 表示处在第i阶并且上一轮跳了j阶 (j = 1 或 2)
    dp = [[0] * 3 for _ in range(n + 1)]
    # 初始状态
    dp[1][1], dp[1][2] = 1, 0
    dp[2][1], dp[2][2] = 0, 1
    
    # 状态转移
    for i in range(3, n + 1):
        dp[i][1] = dp[i - 1][2]          # 只能从dp[i-1][2]转移来
        dp[i][2] = dp[i - 2][1] + dp[i - 2][2]  # 可以从dp[i-2][1]或dp[i-2][2]转移来
    
    return dp[n][1] + dp[n][2]
```

## 14.3 动态规划解题思路

### 问题判断

**适合动态规划的问题特征**：
1. 包含重叠子问题
2. 具有最优子结构
3. 满足无后效性

**加分项**：
- 问题包含最大(小)或最多(少)等最优化描述
- 状态能够用列表、矩阵或树来表示
- 状态与其周围状态存在递推关系

**减分项**：
- 目标是找出所有可能的解决方案
- 有明显的排列组合特征

### 解题步骤

以**最小路径和**问题为例：

**问题**：给定 $n \times m$ 的二维网格grid，机器人从左上角到右下角的最小路径和。

#### 第一步：定义状态

```python
# 状态[i][j] = 从[0][0]走到[i][j]的最小路径和
# dp表尺寸与grid相同
```

#### 第二步：推导状态转移方程

由于只能从上方或左方到达当前格子：

$$dp[i][j] = \min(dp[i-1][j], dp[i][j-1]) + grid[i][j]$$

#### 第三步：确定边界条件和转移顺序

```python
def min_path_sum(grid: List[List[int]]) -> int:
    """最小路径和"""
    m, n = len(grid), len(grid[0])
    
    # 初始化dp表
    dp = [[0] * n for _ in range(m)]
    dp[0][0] = grid[0][0]
    
    # 首行只能从左边来
    for j in range(1, n):
        dp[0][j] = dp[0][j - 1] + grid[0][j]
    
    # 首列只能从上边来
    for i in range(1, m):
        dp[i][0] = dp[i - 1][0] + grid[i][0]
    
    # 状态转移：遍历其余格子
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]
    
    return dp[m - 1][n - 1]
```

**空间优化**：由于 $dp[i][j]$ 只与 $dp[i-1][j]$ 和 $dp[i][j-1]$ 有关，可以用一行数组优化：

```python
def min_path_sum_comp(grid: List[List[int]]) -> int:
    """最小路径和：空间优化"""
    m, n = len(grid), len(grid[0])
    
    # 用一行数组
    dp = [0] * n
    for i in range(m):
        for j in range(n):
            if i == 0 and j == 0:
                dp[j] = grid[i][j]
            elif i == 0:
                dp[j] = dp[j - 1] + grid[i][j]
            elif j == 0:
                dp[j] = dp[j] + grid[i][j]
            else:
                dp[j] = min(dp[j], dp[j - 1]) + grid[i][j]
    
    return dp[n - 1]
```

## 经典问题汇总

### 1. 0-1背包问题

**问题**：给定n个物品，第i个物品的重量为wgt[i-1]、价值为val[i-1]，和一个容量为cap的背包，每个物品只能选择一次，求背包能装下的最大价值。

**状态定义**：`dp[i][c]` = 前i个物品在容量c下的最大价值

**状态转移**：
$$dp[i][c] = \max(dp[i-1][c], dp[i-1][c-wgt[i-1]] + val[i-1])$$

**空间优化**：
```python
def knapsack_dp(wgt: List[int], val: List[int], cap: int) -> int:
    """0-1背包问题"""
    n = len(wgt)
    dp = [0] * (cap + 1)
    
    for i in range(n):
        # 倒序遍历保证每个物品只使用一次
        for c in range(cap, wgt[i] - 1, -1):
            dp[c] = max(dp[c], dp[c - wgt[i]] + val[i])
    
    return dp[cap]
```

### 2. 完全背包问题

**问题**：每种物品可以重复选取。

**关键区别**：正序遍历容量
```python
def complete_knapsack(wgt: List[int], val: List[int], cap: int) -> int:
    """完全背包问题"""
    n = len(wgt)
    dp = [0] * (cap + 1)
    
    for i in range(n):
        # 正序遍历允许多次使用同一物品
        for c in range(wgt[i], cap + 1):
            dp[c] = max(dp[c], dp[c - wgt[i]] + val[i])
    
    return dp[cap]
```

### 3. 编辑距离

**问题**：从一个字符串到另一个字符串的最少编辑步数（添加、删除、替换）。

**状态定义**：`dp[i][j]` = 将s的前i个字符转换为t的前j个字符所需的最少步数

**状态转移**：
- 当 `s[i-1] == t[j-1]`：`dp[i][j] = dp[i-1][j-1]`
- 否则：`dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1`

## 总结

| 动态规划类型 | 状态转移顺序 | 空间优化 |
|-------------|-------------|---------|
| 0-1背包 | 倒序遍历 | 一维数组 |
| 完全背包 | 正序遍历 | 一维数组 |
| 编辑距离 | 二维遍历 | 可优化 |

**核心思想**：空间换时间，通过存储子问题的解避免重复计算。
