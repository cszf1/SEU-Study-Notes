## 1.1 算法无处不在

算法并非遥不可及的高深知识，我们在日常生活中已经习惯使用各种算法。

### 例一：查字典 → 二分查找

**场景**：在按拼音排序的字典中查找首字母为 `r` 的字

**步骤**：
1. 翻开字典约一半的页数，查看首字母（如 `m`）
2. 因为 `r` 在拼音字母表中位于 `m` 之后，排除前半部分
3. 不断重复，最终找到目标页码

**本质**：这就是著名的 **二分查找（Binary Search）** 算法。将字典视为一个已排序的"数组"，一系列操作就是二分查找。

### 例二：整理扑克 → 插入排序

**场景**：将手中的扑克牌从小到大排序

**步骤**：
1. 将扑克牌划分为"有序"和"无序"两部分，初始时最左1张已有序
2. 从无序部分抽出一张牌，插入至有序部分的正确位置
3. 循环操作，直至所有扑克牌有序

**本质**：这就是 **插入排序（Insertion Sort）** 算法，在处理小型数据集时非常高效。

### 例三：货币找零 → 贪心算法

**场景**：购买69元商品，支付100元，需找零31元

**步骤**：
1. 可选货币：1元、5元、10元、20元
2. 优先选择最大面额的币种
3. 逐步减少剩余金额：31 → 11 → 1 → 0
4. 最终方案：20 + 10 + 1 = 31元

**本质**：这就是 **贪心算法（Greedy Algorithm）**。每一步都采取当前看来最好的选择（尽可能用大面额货币）。

### 生活启示

> 从烹饪一道菜到星际航行，几乎所有问题的解决都离不开算法。计算机使得我们能够将生活中的问题转移到计算机上，以更高效的方式解决各种复杂问题。

## 1.2 算法是什么

### 1.2.1 算法定义

**算法（Algorithm）**：在有限时间内解决特定问题的一组指令或操作步骤。

**特性**：
- **明确性**：问题是明确的，包含清晰的输入和输出定义
- **可行性**：能够在有限步骤、时间和内存空间下完成
- **确定性**：各步骤都有确定的含义，相同输入产生相同输出

### 1.2.2 数据结构定义

**数据结构（Data Structure）**：组织和存储数据的方式。

**设计目标**：
- 空间占用尽量少，节省计算机内存
- 数据操作尽可能快速（访问、添加、删除、更新等）
- 提供简洁的数据表示和逻辑信息

**权衡原则**：
| 对比 | 优势 | 劣势 |
|------|------|------|
| 链表 vs 数组 | 添加/删除更便捷 | 数据访问速度慢 |
| 图 vs 链表 | 提供更丰富的逻辑信息 | 占用更大内存空间 |

### 1.2.3 数据结构与算法的关系

```
数据结构 ─────┐
              ├──→ 紧密结合
算法     ─────┘
```

**三个方面**：
1. **数据结构是算法的基石**：为算法提供结构化存储的数据，以及操作数据的方法
2. **算法为数据结构注入生命力**：数据结构本身仅存储数据，结合算法才能解决特定问题
3. **算法可基于不同数据结构实现**：执行效率可能相差很大，选择合适的数据结构是关键

### 积木类比

| 数据结构与算法 | 拼装积木 |
|---|---|
| 输入数据 | 未拼装的积木 |
| 数据结构 | 积木组织形式（形状、大小、连接方式等） |
| 算法 | 把积木拼成目标形态的一系列操作步骤 |
| 输出数据 | 积木模型 |

### 约定俗成的简称

> 在实际讨论时，我们通常将"数据结构与算法"简称为"算法"。比如众所周知的 LeetCode 算法题目，实际上同时考查数据结构和算法两方面的知识。

## 1.3 小结

### 重点回顾

1. **算法在日常生活中无处不在**：查字典、整理扑克、货币找零等日常活动都蕴含着算法思想

2. **常见算法对应关系**：
   - 查字典 → 二分查找（分而治之思想）
   - 整理扑克 → 插入排序（适合小型数据集）
   - 货币找零 → 贪心算法（每步最优选择）

3. **算法定义**：在有限时间内解决特定问题的一组指令或操作步骤

4. **数据结构定义**：计算机中组织和存储数据的方式

5. **数据结构与算法的关系**：
   - 数据结构是算法的基石
   - 算法为数据结构注入生命力
   - 两者紧密结合，类比为拼装积木

### Q & A 精选

**Q：工作中常用算法都被编程语言封装好了，我们还需要学习算法吗？**

如果把具体的工作技能比作是武功的"招式"，基础科目应该更像是"内功"。

学算法的意义：
- 不是从零实现它，而是作出**专业的反应和判断**
- 例如：Python 内置排序函数时间复杂度是 $O(n \log n)$
- 但如果数据是固定位数的整数（如学号），可以用**基数排序**将复杂度降为 $O(nk)$
- 当数据体量很大时，节省的时间就能创造较大价值

> **人的知识越完备、经验越多，分析问题就会越深入，问题就能被解决得更优雅。**

## 代码示例

### Python 实现示例

```python
# 二分查找（查字典算法）
def binary_search字典(words: list, target: str) -> int:
    """在有序字典中查找目标词"""
    left, right = 0, len(words) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if words[mid] == target:
            return mid
        elif words[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1  # 未找到

# 插入排序（整理扑克算法）
def insertion_sort(poker: list) -> list:
    """插入排序"""
    for i in range(1, len(poker)):
        key = poker[i]
        j = i - 1
        while j >= 0 and poker[j] > key:
            poker[j + 1] = poker[j]
            j -= 1
        poker[j + 1] = key
    return poker

# 贪心算法（货币找零）
def greedy_change(amount: int) -> dict:
    """贪心算法找零"""
    coins = [20, 10, 5, 1]  # 可用面额
    result = {}
    
    for coin in coins:
        if amount >= coin:
            count = amount // coin
            result[coin] = count
            amount -= coin * count
    
    return result  # 例如: {20: 1, 10: 1, 1: 1} = 31元
```
