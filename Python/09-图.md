# 第9章 图（Graph）

> 在生命旅途中，我们就像是一个个节点，被无数看不见的边相连。每一次的相识与相离，都在这张巨大的网络图中留下独特的印记。

## 9.1 图

### 定义

图（graph）是一种非线性数据结构，由**顶点（vertex）**和**边（edge）**组成。图 $G$ 可以抽象表示为：

$$G = \{V, E\}$$

其中：
- $V$：顶点的集合
- $E$：边的集合

图与链表、树的关系：顶点看作节点，边看作连接各个节点的引用（指针）。链表和树都是图的特例，网络关系（图）的自由度更高，因而更为复杂。

### 图的常见类型

| 分类标准 | 类型 | 说明 |
|---------|------|------|
| **边方向** | 无向图 | 边表示"双向"连接，如微信好友关系 |
| | 有向图 | 边具有方向性，如微博关注关系 |
| **连通性** | 连通图 | 从某顶点可达其余所有顶点 |
| | 非连通图 | 从某顶点出发，至少有一个顶点无法到达 |
| **权重** | 有权图 | 每条边包含权重变量 |
| | 无权图 | 边无权重 |

### 图的术语

- **邻接（adjacency）**：两顶点之间存在边相连
- **路径（path）**：从顶点 A 到顶点 B 经过的边构成的序列
- **度（degree）**：
  - 入度（in-degree）：有多少条边指向该顶点
  - 出度（out-degree）：有多少条边从该顶点指出

### 图的表示方法

#### 1. 邻接矩阵

使用 $n \times n$ 大小的矩阵表示图，行列代表顶点，矩阵元素代表边（1=有边，0=无边）。

**特性**：
- 简单图中，主对角线元素无意义
- 无向图的邻接矩阵关于主对角线对称
- 空间复杂度：$O(n^2)$

**优点**：增删查改操作效率高（$O(1)$）

**缺点**：空间占用多

#### 2. 邻接表

使用 $n$ 个链表表示图，第 $i$ 个链表存储该顶点的所有邻接顶点。

**特性**：
- 只存储实际存在的边（通常 $m \ll n^2$）
- 空间复杂度：$O(n + m)$

**优点**：节省空间

**缺点**：查找边需要遍历链表（$O(n)$）

**优化**：链表过长时可转化为 AVL 树/红黑树（$O(\log n)$）或哈希表（$O(1)$）

### 图的常见应用

| 场景 | 顶点 | 边 | 图计算问题 |
|------|------|-----|-----------|
| 社交网络 | 用户 | 好友关系 | 潜在好友推荐 |
| 地铁线路 | 站点 | 连通性 | 最短路线推荐 |
| 太阳系 | 星体 | 万有引力 | 行星轨道计算 |

## 9.2 图的基础操作

### 邻接矩阵实现

```python
class GraphAdjMat:
    """基于邻接矩阵实现的无向图类"""
    
    def __init__(self, vertices: list[int], edges: list[list[int]]):
        """构造方法"""
        self.vertices: list[int] = []      # 顶点列表
        self.adj_mat: list[list[int]] = [] # 邻接矩阵
        
        for val in vertices:
            self.add_vertex(val)
        for e in edges:
            self.add_edge(e[0], e[1])
    
    def size(self) -> int:
        return len(self.vertices)
    
    def add_vertex(self, val: int):
        """添加顶点"""
        n = self.size()
        self.vertices.append(val)
        new_row = [0] * n
        self.adj_mat.append(new_row)
        for row in self.adj_mat:
            row.append(0)
    
    def remove_vertex(self, index: int):
        """删除顶点"""
        if index >= self.size():
            raise IndexError()
        self.vertices.pop(index)
        self.adj_mat.pop(index)
        for row in self.adj_mat:
            row.pop(index)
    
    def add_edge(self, i: int, j: int):
        """添加边"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError()
        self.adj_mat[i][j] = 1
        self.adj_mat[j][i] = 1  # 无向图对称
    
    def remove_edge(self, i: int, j: int):
        """删除边"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError()
        self.adj_mat[i][j] = 0
        self.adj_mat[j][i] = 0
```

### 邻接表实现

```python
from collections import defaultdict, deque

class Vertex:
    """顶点类"""
    def __init__(self, val: int):
        self.val = val
    
    def __hash__(self):
        return hash(id(self))
    
    def __eq__(self, other):
        return self is other

class GraphAdjList:
    """基于邻接表实现的无向图类"""
    
    def __init__(self, edges: list[list[Vertex]]):
        self.adj_list = dict[Vertex, list[Vertex]]()  # 邻接表
        
        for edge in edges:
            self.add_vertex(edge[0])
            self.add_vertex(edge[1])
            self.add_edge(edge[0], edge[1])
    
    def size(self) -> int:
        return len(self.adj_list)
    
    def add_vertex(self, vet: Vertex):
        """添加顶点"""
        if vet in self.adj_list:
            return
        self.adj_list[vet] = []
    
    def remove_vertex(self, vet: Vertex):
        """删除顶点"""
        if vet not in self.adj_list:
            raise ValueError()
        self.adj_list.pop(vet)
        for vertex in self.adj_list:
            if vet in self.adj_list[vertex]:
                self.adj_list[vertex].remove(vet)
    
    def add_edge(self, vet1: Vertex, vet2: Vertex):
        """添加边"""
        if vet1 not in self.adj_list or vet2 not in self.adj_list or vet1 == vet2:
            raise ValueError()
        self.adj_list[vet1].append(vet2)
        self.adj_list[vet2].append(vet1)
    
    def remove_edge(self, vet1: Vertex, vet2: Vertex):
        """删除边"""
        if vet1 not in self.adj_list or vet2 not in self.adj_list or vet1 == vet2:
            raise ValueError()
        self.adj_list[vet1].remove(vet2)
        self.adj_list[vet2].remove(vet1)
```

### 效率对比

| 操作 | 邻接矩阵 | 邻接表（链表） | 邻接表（哈希表） |
|------|---------|---------------|----------------|
| 判断是否邻接 | $O(1)$ | $O(n)$ | $O(1)$ |
| 添加边 | $O(1)$ | $O(1)$ | $O(1)$ |
| 删除边 | $O(1)$ | $O(n)$ | $O(1)$ |
| 添加顶点 | $O(n)$ | $O(1)$ | $O(1)$ |
| 删除顶点 | $O(n^2)$ | $O(n + m)$ | $O(n)$ |
| 空间复杂度 | $O(n^2)$ | $O(n + m)$ | $O(n + m)$ |

**总结**：
- 邻接矩阵：以空间换时间
- 邻接表：以时间换空间

## 9.3 图的遍历

树是图的特例，树的遍历也是图的遍历的特例。图的遍历方式有两种：**广度优先遍历（BFS）** 和 **深度优先遍历（DFS）**。

### 广度优先遍历（BFS）

**思想**：由近及远、层层扩张，优先访问距离最近的顶点。

**实现**：借助队列（先入先出）。

```python
from collections import deque

def graph_bfs(graph: GraphAdjList, start_vet: Vertex) -> list[Vertex]:
    """广度优先遍历"""
    res = []                      # 顶点遍历序列
    visited = set[Vertex]([start_vet])  # 已访问顶点集合
    que = deque[Vertex]([start_vet])     # 队列
    
    while len(que) > 0:
        vet = que.popleft()      # 队首顶点出队
        res.append(vet)          # 记录访问
        
        for adj_vet in graph.adj_list[vet]:
            if adj_vet in visited:
                continue
            que.append(adj_vet)
            visited.add(adj_vet)
    
    return res
```

**复杂度分析**：
- 时间复杂度：$O(|V| + |E|)$（所有顶点入队出队一次，所有边被访问2次）
- 空间复杂度：$O(|V|)$（res、visited、que 最多存 |V| 个顶点）

**说明**：BFS 序列不唯一，相同距离的顶点顺序可以任意交换。

### 深度优先遍历（DFS）

**思想**：优先走到底、无路可走再回头的遍历方式。

**实现**：基于递归（回溯）。

```python
def graph_dfs(graph: GraphAdjList, start_vet: Vertex) -> list[Vertex]:
    """深度优先遍历"""
    res = []
    visited = set[Vertex]()
    
    def dfs(graph: GraphAdjList, visited: set[Vertex], res: list[Vertex], vet: Vertex):
        res.append(vet)
        visited.add(vet)
        for adj_vet in graph.adj_list[vet]:
            if adj_vet not in visited:
                dfs(graph, visited, res, adj_vet)
    
    dfs(graph, visited, res, start_vet)
    return res
```

**复杂度分析**：
- 时间复杂度：$O(|V| + |E|)$
- 空间复杂度：$O(|V|)$（res、visited、递归深度）

**说明**：DFS 序列也不唯一，邻接顶点的探索顺序可以任意选择。

## 9.4 小结

### 重点回顾

1. **图由顶点和边组成**，可以表示为 $\{V, E\}$ 集合
2. **图的类型**：
   - 有向图 / 无向图
   - 连通图 / 非连通图
   - 有权图 / 无权图
3. **图的表示**：
   - 邻接矩阵：$O(1)$ 增删查改，$O(n^2)$ 空间
   - 邻接表：$O(n)$ 查找，$O(n+m)$ 空间
4. **遍历算法**：
   - BFS（广度优先）：队列实现，由近及远
   - DFS（深度优先）：递归实现，走到底再回溯
5. **应用场景**：社交网络、地图导航、知识图谱等

### 与C++对比

| 特性 | Python | C++ |
|------|--------|-----|
| 邻接表 | `dict[Vertex, list]` | `unordered_map + vector` |
| 顶点标识 | 对象实例（唯一性） | 节点指针或索引 |
| 队列 | `collections.deque` | `queue` |
| 集合 | `set` | `unordered_set` |
