# 第15章 贪心

## 本章导览

向日葵朝着太阳转动，时刻追求自身成长的最大可能。贪心策略在一轮轮的简单选择中，逐步导向最佳答案。

## 15.1 贪心算法

### 基本概念

**贪心算法（Greedy Algorithm）** 是一种常见的解决优化问题的算法，其基本思想是：在问题的每个决策阶段，都选择当前看起来最优的选择，即贪心地做出局部最优的决策，以期获得全局最优解。

### 贪心 vs 动态规划

| 特征 | 贪心算法 | 动态规划 |
|------|----------|----------|
| 考虑历史决策 | 不考虑，一路向前 | 考虑之前所有决策 |
| 工作方式 | 不断贪心选择，缩小问题范围 | 使用过去子问题的解构建当前解 |
| 时间复杂度 | 通常更低 | 通常较高 |

### 例题：零钱兑换

**问题**：给定 n 种硬币，第 i 种硬币的面值为 `coins[i-1]`，目标金额为 `amt`，每种硬币可以重复选取，问能够凑出目标金额的最少硬币数量。如果无法凑出目标金额，则返回 -1。

**贪心策略**：给定目标金额，贪心地选择不大于且最接近它的硬币，不断循环该步骤，直至凑出目标金额为止。

```python
from typing import List

def coin_change_greedy(coins: List[int], amt: int) -> int:
    """零钱兑换：贪心"""
    # 假设coins列表有序
    i = len(coins) - 1
    count = 0
    
    # 循环进行贪心选择，直到无剩余金额
    while amt > 0:
        # 找到小于且最接近剩余金额的硬币
        while i > 0 and coins[i] > amt:
            i -= 1
        # 选择coins[i]
        amt -= coins[i]
        count += 1
    
    # 若未找到可行方案，则返回
    return count if amt == 0 else -1
```

### 贪心算法的优点与局限

**优点**：
- 操作直接、实现简单
- 时间复杂度低：`O(amt / min(coins))`，远优于动态规划的 `O(n × amt)`

**局限**：贪心算法无法保证找到全局最优解。

**反例**：
- `coins = [1, 20, 50]`，`amt = 60`：贪心得到 50+1×10 = 11枚，最优解 20+20+20 = 3枚
- `coins = [1, 49, 50]`，`amt = 98`：贪心得到 50+1×48 = 49枚，最优解 49+49 = 2枚

### 适用情况

1. **可以保证找到最优解**：贪心算法往往是最优选择，比回溯、动态规划更高效
2. **可以找到近似最优解**：对于复杂问题，能找到次优解也是不错的

### 贪心算法特性

**两个关键性质**：
1. **贪心选择性质**：局部最优选择始终可以导致全局最优解
2. **最优子结构**：原问题的最优解包含子问题的最优解

### 贪心算法解题步骤

1. **问题分析**：梳理状态定义、优化目标和约束条件
2. **确定贪心策略**：确定如何在每一步做出贪心选择
3. **正确性证明**：证明问题具有贪心选择性质和最优子结构

### 典型应用

- 硬币找零问题（某些硬币组合下）
- 区间调度问题（选择结束时间最早的任务）
- 分数背包问题（选择性价比最高的物品）
- 股票买卖问题
- 霍夫曼编码
- Dijkstra 算法

## 15.2 分数背包问题

### 问题描述

给定 n 个物品，第 i 个物品的重量为 `wgt[i-1]`，价值为 `val[i-1]`，和一个容量为 cap 的背包。每个物品只能选择一次，但可以选择物品的一部分，问在限定背包容量下背包中物品的最大价值。

### 与0-1背包的区别

| 特征 | 0-1背包 | 分数背包 |
|------|---------|----------|
| 选择方式 | 整个物品或不选 | 可选择物品的一部分 |
| 贪心适用 | 否 | 是 |

### 贪心策略

**核心思想**：最大化单位重量下的物品价值。

1. 将物品按照单位价值（`val[i-1]/wgt[i-1]`）从高到低排序
2. 每轮贪心地选择单位价值最高的物品
3. 若剩余容量不足，则使用当前物品的一部分填满背包

### 代码实现

```python
from typing import List

class Item:
    """物品"""
    def __init__(self, w: int, v: int):
        self.w = w  # 物品重量
        self.v = v  # 物品价值

def fractional_knapsack(wgt: List[int], val: List[int], cap: int) -> int:
    """分数背包：贪心"""
    # 创建物品列表，包含两个属性：重量、价值
    items = [Item(w, v) for w, v in zip(wgt, val)]
    
    # 按照单位价值 item.v/item.w 从高到低进行排序
    items.sort(key=lambda item: item.v / item.w, reverse=True)
    
    # 循环贪心选择
    res = 0
    for item in items:
        if item.w <= cap:  # 若剩余容量充足，则将当前物品整个装进背包
            res += item.v
            cap -= item.w
        else:  # 若剩余容量不足，则将当前物品的一部分装进背包
            res += (item.v / item.w) * cap
            break  # 已无剩余容量，因此跳出循环
    
    return res
```

### 复杂度分析

- **时间复杂度**：O(n log n)（排序）+ O(n)（遍历）= O(n log n)
- **空间复杂度**：O(n)

### 正确性证明（反证法）

假设物品 a 是单位价值最高的物品，使用某算法求得最大价值为 res，但该解中不包含物品 a。

从背包中拿出单位重量的任意物品，并替换为单位重量的物品 a。由于物品 a 的单位价值最高，因此替换后的总价值一定大于 res。这与 res 是最优解矛盾，说明最优解中必须包含物品 a。

## 15.3 最大容量问题

### 问题描述

输入一个数组 `ht`，每个元素代表一个垂直隔板的高度。数组中的任意两个隔板以及它们之间的空间可以组成一个容器。

- **容量（面积）** = 高度 × 宽度
- **高度** = 较短的隔板决定
- **宽度** = 两个隔板的数组索引之差

### 数学表达

```
cap[i,j] = min(ht[i], ht[j]) × (j - i)
```

### 贪心策略

**核心洞察**：
- 在状态 `[i, j]` 下，若 i 为短板、j 为长板
- 向内移动长板 → 容量一定变小（宽度变小，高度不变或变小）
- 向内移动短板 → 可能使容量变大（宽度变小，但高度可能变大）

**贪心策略**：初始化两指针，使其分列容器两端，每轮向内收缩短板对应的指针，直至两指针相遇。

### 代码实现

```python
from typing import List

def max_capacity(ht: List[int]) -> int:
    """最大容量：贪心"""
    # 初始化i,j，使其分列数组两端
    i, j = 0, len(ht) - 1
    # 初始最大容量为0
    res = 0
    
    # 循环贪心选择，直至两板相遇
    while i < j:
        # 更新最大容量
        cap = min(ht[i], ht[j]) * (j - i)
        res = max(res, cap)
        
        # 向内移动短板
        if ht[i] < ht[j]:
            i += 1
        else:
            j -= 1
    
    return res
```

### 复杂度分析

- **时间复杂度**：O(n)
- **空间复杂度**：O(1)

### 正确性证明

在状态 `cap[i,j]` 下，若 i 为短板、j 为长板，贪心地将短板 i 向内移动，会"跳过"以下状态：
```
cap[i,i+1], cap[i,i+2], ..., cap[i,j-2], cap[i,j-1]
```

这些被跳过的状态实际上是将长板 j 向内移动的所有状态。由于内移长板一定会导致容量变小，因此被跳过的状态都不可能是最优解，跳过它们不会导致错过最优解。

## 15.4 最大切分乘积问题

### 问题描述

给定一个正整数 n，将其切分为至少两个正整数的和，求切分后所有整数的乘积最大是多少。

### 数学表达

将 n 切分为 m 个整数因子，其中第 k 个因子记为 n_k：

```
n = Σ(n_k)，k = 1, 2, ..., m
目标：max(Π(n_k))
```

### 贪心策略推导

**第一步**：证明大于等于 4 的整数都应该被切分。

假设从 n 中分出一个因子 2，则乘积为 2(n-2)：
```
2(n-2) ≥ n
→ 2n - 4 ≥ n
→ n ≥ 4
```

因此，当 n ≥ 4 时，切分出一个 2 后乘积会变大。

**第二步**：确定最优切分因子。

在 1、2、3 三个因子中：
- 1 是最差的（1×(n-1) < n 恒成立）
- 当 n = 6 时，3×3 > 2×2×2，所以 3 比 2 更优
- 最多只应存在两个 2，因为三个 2 可以替换为两个 3

**贪心策略**：
1. 不断地切分出因子 3，直至余数为 0、1、2
2. 当余数为 0 时：不做任何处理
3. 当余数为 2 时：保留
4. 当余数为 1 时：将最后一个 3 替换为 2（因为 2×2 > 1×3）

### 代码实现

```python
import math
from typing import List

def max_product_cutting(n: int) -> int:
    """最大切分乘积：贪心"""
    # 当 n <= 3 时，必须切分出一个 1
    if n <= 3:
        return 1 * (n - 1)
    
    # 贪心地切分出 3，a 为 3 的个数，b 为余数
    a = n // 3
    b = n % 3
    
    # 当余数为 1 时，将一对 1*3 转化为 2*2
    if b == 1:
        return int(math.pow(3, a - 1)) * 2 * 2
    # 当余数为 2 时，不做处理
    if b == 2:
        return int(math.pow(3, a)) * 2
    # 当余数为 0 时，不做处理
    return int(math.pow(3, a))
```

### 复杂度分析

- **时间复杂度**：O(1)（取决于幂运算实现）
- **空间复杂度**：O(1)

### 正确性证明（反证法）

1. **所有因子 < 3**：假设最优方案中存在 > 4 的因子，将其继续划分为 2(a-2)，乘积更大或相等，矛盾
2. **不包含 1**：假设有因子 1，合并入另一因子可获得更大乘积，矛盾
3. **最多包含两个 2**：三个 2 替换为两个 3，乘积更大，矛盾

## 15.5 小结

### 重点回顾

1. **贪心算法原理**：在每个决策阶段都做出局部最优的决策，以期获得全局最优解
2. **迭代特性**：每轮将问题转化成一个规模更小的子问题，直到问题被解决
3. **效率优势**：实现简单，时间复杂度通常比动态规划更低
4. **适用条件**：具有贪心选择性质和最优子结构
5. **注意事项**：对于某些问题（如零钱兑换），贪心可能找到很差的解

### 各问题对比

| 问题 | 贪心策略 | 时间复杂度 | 能否保证最优 |
|------|----------|------------|--------------|
| 零钱兑换 | 选择最接近的硬币 | O(amt/min) | 不一定 |
| 分数背包 | 选择最高单位价值 | O(n log n) | 能 |
| 最大容量 | 移动短板 | O(n) | 能 |
| 最大切分乘积 | 切分出3 | O(1) | 能 |
